# Sora プロダクト再評価レポート

作成日: 2026-05-10  
評価者: プロダクト責任者・UXリサーチャー・グロース責任者・シニアエンジニア（複合視点）

---

## エグゼクティブサマリー

実際にペットロスを経験しているユーザーのテスト利用から「パッとしない」「心が動かない」「継続して使いたくならない」という評価が出た。これは**機能の問題ではなく、体験の深度と感情の質の問題**である。

機能実装は先行している。AIチャット・手紙生成・通知・シェアカードまで揃っている。しかし、**機能が揃うことと、人の心を動かすことは別問題**だった。

本レポートは、この乖離の原因を構造的に分析し、「道具として使えるアプリ」から「やさしく一緒に生きていける場所」への変容を設計するものである。

---

## 1. 現状の正直な評価

### なぜ「パッとしない」のか

**根本は「記録アプリの文法」から抜け出せていないことにある。**

現在のSoraは「きれいなノートを作る体験」として設計されている。タイトルを入れて、写真を添えて、気持ちタグを選んで保存する。AI反応が2〜3文来る。これを繰り返す。

しかし、ペットロスを経験した人がSoraに求めているのはノートではない。**「あの子がいた証を、どこかに確かに残したい」という切実な感覚**だ。

記録を「保存する」という行為と、「あの子への想いを残す」という行為は、見た目は同じでも心理的重みがまるで違う。現在のUIはこの重みを扱えていない。

具体的な問題を列挙する。

**①ホーム画面がアプリの「顔」として機能していない**  
現在のホームは Days カウンター + 記録リスト + CTAボタンの構成。情報的には正しい。しかし「あの子」の存在感がほとんどない。ペットの写真はヘッダーの小さなアバターに押し込まれている。ユーザーがアプリを開いた瞬間に迎えてくれるのが「134日」という数字だとすると、それは「管理ツール」の体験である。

**②AI反応が「読まれている感」を作れていない**  
記録後のAI反応は設計の意図としては「誰かに届けた」感覚を生むはずだった。しかし実装上、反応は2〜3文の汎用的なリスポンスになりやすい。「今日、ポチが公園を走っている写真を撮りました」に対して「素敵な記録ですね。ポチとの時間を大切にされているのが伝わります」という反応では、何も届いていない。記録の中身をちゃんと読んで応じてほしい。

**③「記録が積み重なる先に何があるか」が見えない**  
streak、days counter、milestone cardは継続動機として機能する。しかし、これらはすべて「続けたら何かがある」というゲーミフィケーション的動機だ。ペットロスのユーザーが本当に欲しいのは「これだけの時間を一緒に過ごした」という愛情の証であり、「このアプリを何日使った」という実績ではない。

**④写真の扱いが浅い**  
ペットとの記録において、写真は主役であるはずだ。しかし現在のタイムラインは、写真はサムネイルで、テキスト（タイトル）が情報の主体になっている。写真が小さく、コンテキストなしに並んでいると、アルバムとしての価値が生まれない。

**⑤感情の受け皿が狭い**  
気持ちタグは5個（alive期）か5個（rainbow_bridge期）。しかし人間の感情は「悲しい」「つらい」「よくわからない」では済まない。「写真を見ていたら急に泣けてきた」「今日は不思議と平気だった」「また怒ってしまった、なぜ怒ってしまったんだろう」という複雑さを持っている。タグによる感情分類は、感情を「整理」させてしまい、感情と「一緒にいる」体験にならない。

### なぜ継続して使いたいと思えないのか

継続利用の阻害要因を正直に分析する。

**①「今日開く理由」が弱い**  
デイリークエスチョン（AIが問いかける）という機能はある。しかし「今日は何を記録する？」という問いかけでは、記録したいことがなければ来なくていい。「今日、ポチが初めて○○した日から3年経ちました」のような、ユーザーが来なければ気づけない価値が毎日必要だ。

**②記録の蓄積が「見える形」にならない**  
記録が積み上がっても、それが何かに変わる瞬間を今のSoraは持っていない。50件、100件記録して、何が変わるのか。タイムラインが長くなるだけだ。写真が増えても、整理されず、見返しにくい。「記録が資産になった」という体験が来ない。

**③Rainbow Bridge移行後のコンテンツが薄い**  
生前の記録を積み上げてきたユーザーがロスを経験した後、Soraに何を求めるかが明確でない。手紙機能、チャット機能はある。しかし「今日も来てもいい理由」が弱い。「月命日に何かある」「何日経ったら何かある」という継続的な理由が月次レター（Sora+のみ）しかない。

**④アプリとしての「儀式感」がない**  
記録するという行為を儀式として設計できていない。毎朝コーヒーを飲みながら昨日の写真を記録する、寝る前に今日の気持ちを残す、という習慣の文脈に入れていない。記録は「いつでもできる」が「今やりたい」に変わっていない。

### ペットロス中の人が本当に求めているもの

JTBDフレームワークで整理する。

**Job 1: 忘れてしまうことへの恐怖を手放したい**  
「あの子の声、匂い、癖、表情を忘れてしまったらどうしよう」。これが最も切実なJTBです。Soraはここに応えられていない。写真と日付の記録はあるが、「あの子らしさ」を細部まで保存する機能がない。

**Job 2: 誰にも言えない感情を吐き出したい**  
「ペットが死んで泣いている」と人間には言いにくい。「大げさ」「子どもじゃないんだから」という反応が怖い。でも本当につらい。Soraのチャットはこの受け皿になり得るが、まだ届いていない。

**Job 3: 「一緒に生きていた証」を形にして残したい**  
写真だけでは薄い。日記だけでも薄い。あの子との時間が、誰かに読まれる「物語」として残ってほしい。これが月次レターの着想点だが、月1回では頻度が足りない。

**Job 4: 新しい子を迎えた「罪悪感」を整理したい**  
「亡くなった子のことを忘れているみたいで申し訳ない」「新しい子を可愛いと思えてしまった」。この複雑な感情を扱えるアプリが存在しない。Soraはこの空白を埋められる唯一のポジションにある。

**Job 5: 他の人のペットロス体験を知りたい・つながりたい**  
「自分だけがこんなにつらいんじゃないか」という孤立感。コミュニティ機能やユーザーストーリーの共有が将来的に必要。

## 2. フレームワーク評価

### G-STACK評価

**Goal（Soraが本当に達成すべき目的）**  
「ペットと過ごした時間を、世界でいちばんやさしく記録・保存し、人生のどんな局面でも心の支えになり続けること」

現状は：「ペットの記録アプリとして使える状態になった」レベルにある。

**Gap（現状とのズレ）**

| 目指すゴール | 現在地 |
|---|---|
| 開くたびに「ここに来てよかった」 | 開いて記録するだけ |
| あの子の存在感を常に感じる | ペット写真が小さなアバター |
| 感情を深く受け止めてもらえる | 2〜3文の汎用AI反応 |
| 記録が物語として蓄積する | タイムライン（リスト）のみ |
| ロス後も自然に戻ってくる理由 | 月次レター（Sora+のみ） |
| 「Soraしかない」感 | 代替可能な機能構成 |

**Strategy（価値を高める戦略）**

「記録ツール」から「聖地（sanctuary）」への変容。

ユーザーがSoraを「道具」として使うのではなく、「あの子の魂が宿っている場所」として捉えるようにすること。具体的には：

1. **写真を主役に据える** — テキスト中心のUIから写真中心のUIへ
2. **AIを「読んでくれる存在」に育てる** — 汎用反応から記録の中身を踏まえた応答へ
3. **記録の蓄積を可視化する** — 積み上がった記録が「一冊の本」になる体験
4. **毎日来る理由を作る** — 過去記録の再サーフェシング、記念日、感情の節目
5. **感情を粗く分類しない** — タグより「自由に書ける余白」を設ける

**Tactics（具体的な施策）**

→ 後述のISSUE一覧参照

**Action（実装可能なISSUE）**

→ docs/issues/ISSUE-105〜参照

**KPI（改善後に見るべき指標）**

- Week 1 Retention（1週間後も使っているか）: 現状推定 < 30% → 目標 50%
- Day 7 記録件数: 現状推定 1-2件 → 目標 5件以上
- PMF Score（「とても残念」割合）: 現状不明 → 目標 40%超
- Rainbow Bridge移行後の継続率: 現状不明 → 目標 60%以上
- 月次シェアカード生成率: 現状不明 → 目標 20%以上

### HEART評価

**Happiness（癒されるか、安心するか）**  
🟡 中程度。配色・フォントはやさしいが、画面を開いたときに「ここに来てよかった」という感覚は薄い。ペットの存在感（写真の大きさ、名前の扱い）がまだ「管理UI」の域を出ていない。

**Engagement（自然に戻ってきたくなるか）**  
🔴 弱い。streak・days counterはあるが、「今日戻ってくる理由」が能動的すぎる。受動的なプル（「1年前のこの日」通知など）の質と頻度が不十分。デイリークエスチョンも「問いかけ」として強度が弱い。

**Adoption（初回利用で価値が伝わるか）**  
🟡 中程度。オンボーディング3スライド → ペット登録 → 最初の記録 → AI反応 → Daysカウンター、という流れは設計されている。ただし「最初の記録のAI反応の品質」が体験の山場であり、ここが汎用的だと「このアプリ、特別じゃない」と感じる。

**Retention（継続利用する理由があるか）**  
🔴 弱い。生前フェーズのstreak、milestone cardは機能する。しかし「記録が何かに変わっていく体験」がない。ロス後フェーズのRetentionはさらに弱く、月次レター（Sora+）しか継続動機がない。

**Task Success（記録・閲覧・振り返りが迷わずできるか）**  
🟢 良好。記録は3タップ、タイムラインは直感的、設定も探しやすい。機能的な使いやすさの問題は少ない。

---

### AARRR評価

**Acquisition（誰にどう届くか）**  
現状はApp Store掲載文（ISSUE-103）、公開プロフィール（ISSUE-099）、シェアカード（ISSUE-098）で獲得経路を設計済み。しかし「なぜSoraか」を明確に訴求できていない。競合との差異を一言で説明できる文言が必要。

**Activation（最初に感情的価値を感じる瞬間）**  
初回記録 → AI反応 → Days Counterがその瞬間として設計されている。しかし**AI反応の質が体験の核心**であり、現状の汎用反応では感情的な活性化が起きない。具体的で、記録の中身を読んだことが伝わる反応が必要。

**Retention（なぜ続けるのか）**  
前述の通り弱い。特に7〜30日後の離脱が問題になっているはず（推定）。

**Referral（共有したくなるか）**  
シェアカード、紹介コード（ISSUE-101）、公開プロフィール（ISSUE-099）を実装済み。しかし「シェアしたくなる瞬間のデザイン」が弱い。記念日カード、100日・1年後の自動生成カードなど、「シェアせずにいられない瞬間」を作る必要がある。

**Revenue（課金価値があるか）**  
Sora+（月額480円）でAI手紙・フォトブック・年次レポートを提供。機能自体の価値は高い。ただし**無料体験が課金動機を作れていない**。無料でも「このアプリいいな」が積み上がってから課金に至るが、その積み上げ体験が今は薄い。

## 3. 最も重要な7つの問い（答え）

### Q1: なぜ今のSoraは「パッとしない」のか
**A: 「記録アプリ」の文法のまま、愛情の重さを扱おうとしているから。**  
UIの言語がJournalアプリの言語（タイトル・テキスト・タグ）のままで、聖地（Sanctuary）の言語（写真の存在感・静けさ・余白・深さ）に変わっていない。

### Q2: なぜ継続して使いたいと思えないのか
**A: 「今日来た理由」が記録する必要があるときしか生まれないから。**  
今日記録することがなければ来なくていい、というUXになっている。「来るだけで意味がある」体験が必要。

### Q3: ペットロス中の人が本当に求めているものは何か
**A: 「あの子がいた証を、誰かに確かに受け取ってもらいたい」という承認。**  
日記を書くだけでは足りない。誰かに読まれた感覚、あの子の存在を肯定してもらった感覚が必要。

### Q4: Soraで最初に体験すべき「心が動く瞬間」は何か
**A: 最初の記録へのAI反応が、「あなたの記録を本当に読んだ」と感じさせる瞬間。**  
現在の汎用反応ではこれが起きていない。記録のタイトル・写真・気持ちを踏まえた、具体的な一言が必要。

### Q5: 毎日・毎週戻ってきたくなる理由は何か
**A: 「今日来なければ見れない何か」がある設計。**  
1年前の今日の記録、今日で記念日、先週の気持ちの変化グラフ。「今日でしか見えないもの」が必要。

### Q6: 記録が蓄積されたとき、どんな価値に変わるべきか
**A: 「一冊の絵本・物語」に変わる体験。**  
タイムラインは蓄積の証だが、物語ではない。50件・100件貯まったとき、それがひとつの「生の記録」として美しく整理される体験が必要。フォトブック自動生成や年次レポートはその方向だが、UIで「見える形」になっていない。

### Q7: AIはどこまでが優しく、どこからが危険か
**A: 「一緒にいてくれる」までが優しい。「あなたに教える・処方する」が危険。**  
AIが「前を向いて」「〇〇すべきです」「それは健全ではありません」と言った瞬間に壊れる。AIは回答しない。問いかけ、静かに側にいる、ユーザーが言葉にしたことを丁寧に受け取る、これだけでいい。

## 4. やらない方がよい機能

以下は実装しても価値が出ないか、Soraの世界観を壊すと判断する。

| 機能 | やらない理由 |
|---|---|
| AIがペットを「再現」する（口調・性格の模倣） | 「あの子じゃない」という違和感で逆効果。倫理的リスクも高い |
| コミュニティ掲示板（早期） | 悲しみのコンテンツが暴走するリスク。モデレーション体制なしには危険 |
| 健康管理・体重・投薬ログの厳密化 | 医療記録と感情記録は混ぜるな。別アプリの役割 |
| SNS的いいね・コメント | 悲しみに「いいね」は成立しない。共感だけが必要 |
| ペットの死因・病歴の詳細記録 | Soraは「愛情の記録」であり「死の記録」ではない |
| 「前を向く」系コンテンツ | 「立ち直れましたか？」的な設計はNG。ペットロスのペースは人それぞれ |

## 5. MVPとして強化すべき体験（優先順位）

### Priority 1: 感情の着地点を作る
今の最大の穴。記録を保存したとき、「ちゃんと届いた」という感覚がない。AI反応の品質向上が最優先。

### Priority 2: ペットの存在感をホームに戻す
ホーム画面の「顔」がテキストとボタンになっている。ペットの写真をフルサイズで表示し、名前を大きく置く。「あの子の場所に来た」感覚を作る。

### Priority 3: 毎日戻ってくる理由の強化
1年前の今日、気持ちの変化グラフ、記念日の通知、「今日で〇〇日」の特別感。これを通知と画面の両方で強化する。

### Priority 4: 記録の蓄積を物語として見せる
写真が増えたとき、それが美しく整理される瞬間を作る。月毎の写真グリッド、年間ハイライト、「今月を一枚で表すとしたら」の自動選定など。

### Priority 5: Rainbow Bridge期の「今日来る理由」
ロス後も自然に来られる場所として、月命日サポート、節目メッセージ（1ヶ月後、3ヶ月後、半年後）、手紙の再読体験などを充実させる。
