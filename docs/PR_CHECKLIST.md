# PRチェックリスト

PRを作成・マージする前に全項目を確認する。

---

## ISSUE

- [ ] 対応するISSUEが存在する
- [ ] ISSUEのステータスが `done` に更新されている
- [ ] 1 PR = 1 ISSUE（複数ISSUEをまとめてはいけない）

---

## OpenAPI

- [ ] API変更を伴う場合、`docs/api/openapi.yaml` が先に更新されている
- [ ] 追加・変更したエンドポイントがyamlに定義されている
- [ ] レスポンスの新規フィールドがyamlのスキーマに追記されている
- [ ] yamlに変更がない場合、API変更がないことを確認した

---

## 型・実装

- [ ] `pnpm tsc --noEmit` がエラーなしで通過する
- [ ] フロント手書き型（lib/app-context.tsx）がAPIレスポンスと一致している

---

## テスト

- [ ] `pnpm test` が全件通過する
- [ ] 新機能の正常系テストが追加されている
- [ ] 異常系（バリデーション・権限エラー）のテストが追加されている

---

## 動作確認

- [ ] 実装した機能がブラウザで正常に動作する
- [ ] 変更画面以外のナビゲーションが壊れていない
- [ ] コンソールにエラーが出ていない

---

## コード品質

- [ ] MVPスコープ外の機能を先取り実装していない
- [ ] 複数ISSUEを同時に実装していない
- [ ] 感情設計ルール（CLAUDE.md参照）に違反するコピーがない
