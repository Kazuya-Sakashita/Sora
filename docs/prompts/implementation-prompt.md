# implementation-prompt — ISSUE実装時

ISSUEを実装するときに毎回使うテンプレート。

---

```
ISSUE-XXX を対応してください。

## 対象ISSUE
docs/issues/ISSUE-XXX-title.md を確認してください。

## 実装前にやること（実装前に報告してください）
1. 変更が必要なファイルをリストアップする
2. 既存機能への影響範囲を確認する
3. APIの変更が必要な場合は openapi.yaml を先に更新する
4. 実装方針を1〜2行で示す

確認が終わったら実装に進んでください。

## 実装中のルール
- このISSUEのスコープ外の変更はしない
- 気になる別の問題は新しいISSUEとして切り出す
- 既存のデザインシステム（GlassCard等）を踏襲する
- 感情設計ルール（CLAUDE.md参照）を守る

## 実装後にやること
1. pnpm dev でローカル起動して動作確認する
2. 変更画面の動作確認結果を報告する
3. デグレチェック（オンボーディング→ホームの導線、全タブ）を確認する
4. ISSUEのステータスを done に更新する
5. docs/issues/README.md のテーブルも更新する
```
