# review-prompt — アプリ評価・レビュー時

評価を依頼するときのテンプレート。評価結果はISSUEに変換する。

---

```
Soraアプリ（pnpm dev で起動中）を以下の観点で厳しく評価してください。

## 評価観点

### UX・感情設計
- 最初に開いたとき、何のアプリかが30秒でわかるか
- 生前から使うアプリとして自然に入れるか（ペットロス専用に見えないか）
- 記録の導線が直感的か（3タップ以内で完了できるか）
- 感情設計が適切か（重すぎ・軽すぎ・冷たくないか）
- コピー・文言に問題はないか（NG表現はCLAUDE.md参照）

### 機能・MVP
- MVPに必要な機能（記録・予定・タイムライン・永続化）が揃っているか
- データがリロード後も保持されているか
- 空状態（記録0件）のUIが冷たくないか

### 技術
- コンソールにエラーがないか
- スマートフォンサイズで崩れていないか

## 出力形式

各問題を以下の形式で列挙してください：
- 優先度: critical / high / medium / low
- 問題: （1行）
- 詳細: （2〜3行）
- 影響ファイル: （ファイルパス）

## 評価後のアクション

問題点を docs/issues/ にISSUEとして作成し、
docs/issues/README.md のテーブルに追記してください。
優先度順（criticalが上）に番号を振ってください。
```
