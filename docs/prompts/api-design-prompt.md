# api-design-prompt — API設計・変更時

`docs/api/openapi.yaml` を変更するとき・新しいAPIを追加するときに使う。

---

```
以下の機能についてAPIを設計してください。

## 対象機能
[追加・変更したい機能を書く]

## 現在の openapi.yaml の該当箇所
[変更前の定義、または「新規追加」と書く]

## 設計してほしいこと
- エンドポイント（メソッド + パス）
- リクエストボディのスキーマ
- レスポンスのスキーマ
- エラーレスポンス（application/problem+json 形式）
- 認証の要否

## 制約
- MVPスコープ内に収める
- 既存の openapi.yaml のスタイル・命名規則に合わせる
- 認証はMVP段階では不要（TODOとして残す）
- エラーはRFC 9457のProblemDetail形式に統一

設計が完了したら、openapi.yaml の変更差分を示してください。
実装はまだしないでください。
```
