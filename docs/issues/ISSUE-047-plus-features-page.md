# ISSUE-047: Settings内のSora+機能紹介セクション

name: ISSUE-047
type: feature
priority: Medium
status: done

---

## 概要

Settings 画面の課金UIを「Sora+ でできること」機能紹介セクションとして再設計し、機能認知率と転換率を高める。

## 背景

現状の UpgradeModal は4行の箇条書きのみで、各機能の価値が伝わりにくい。Settings 内で機能ごとに説明 + 「試してみる」ボタンを配置することで、ユーザーが自分に刺さる機能を発見できる。

## 目的

- Plus 機能の認知率向上
- 機能ごとの「使いたい」動機でアップグレードを促す

## スコープ

- `settings.tsx` の課金セクションを以下に再設計（Free ユーザー時のみ）：
  - セクションタイトル：「Sora+ でできること」
  - 機能カード×4（Collapsible なし、常時展開）：

    | 機能 | アイコン | 説明（1行） | ボタン |
    |------|---------|------------|--------|
    | 月別フォトブック | BookOpen | 毎月の思い出をPDFに | 「作ってみる →」 |
    | 年次メモリーレポート | FileText | 1年間の記録を自動まとめ | 「見てみる →」 |
    | 記念日カード保存 | Download | 100日・誕生日を画像に | 「保存してみる →」 |
    | 無制限記録 | Infinity | 50件の制限なし | 「今すぐ始める →」 |

  - 各ボタンタップ → UpgradeModal を表示（`feature` パラメータで開いた機能を渡す）
  - UpgradeModal に `featureName` prop を追加し、タイトルを動的に変える（例：「フォトブックを使うにはSora+が必要です」）
  - 既存の月額/年額ボタンは機能カードの下に移動
- Plus ユーザー時：機能カードをチェックマーク付きで表示（ロック解除済み表示）

## 非スコープ

- 機能ごとのデモ動画
- A/Bテスト

- なし
