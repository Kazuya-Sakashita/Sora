# ISSUE-077: 気持ち記録の時系列リストをfeelingsスクリーンに追加する

name: ISSUE-077
type: improvement
priority: High
status: done

---

## 背景

`feelings.tsx` にはムードトレンドグラフが表示されているが、過去に記録した気持ちの一覧はどこにも存在しない。ユーザーは「あの日どんな気持ちだったか」を振り返ることができない。

気持ちの記録は「感情の軌跡」として機能するとき最も価値を持つ。悲しみのピーク、穏やかになった瞬間、初めて笑えた日——これらが見えることで、ユーザーは自分の回復の過程を自覚できる。記録の意味が実感できないと、feelings機能の継続利用動機が失われる。

## 問題

```tsx
// feelings.tsx — グラフ表示はあるが、個別記録の一覧表示がない
// ユーザーは記録した気持ちを振り返る手段がない
```

## 目的

過去の気持ち記録をシンプルな時系列リストで表示し、「感情の変化」を自分で確認できる体験を作る。

## スコープ

- `feelings.tsx` のムードトレンドグラフ（または現在のUI）の下に時系列リストを追加
- リスト表示内容:
  - 日付（`YYYY年MM月DD日` 形式）
  - タグ絵文字 + タグラベル（例: 🥰 うれしい）
  - メモがある場合はメモ本文（max 50文字 + 省略）
- 並び順: 新しい順
- 最大表示件数: 直近30件（それ以上はスクロールなしで「もっと前の記録を見る」は実装しない）
- 記録が0件の場合は既存の空状態UIを維持
- タグ絵文字のマッピング:
  - happy: 🥰, calm: 😌, fun: 😄, worried: 😟, loving: 💝

## やらないこと

- 気持ち記録の編集・削除
- feelings APIの変更
- フィルタ・検索機能
- グラフデザインの変更

## 受け入れ条件

- [ ] グラフ下に気持ちの時系列リストが表示される
- [ ] 各エントリに日付・絵文字・タグラベル・メモが表示される
- [ ] 記録が0件の場合は空状態UIが表示される
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] 既存のグラフ表示を壊していない

## 確認観点

- ユーザーの心を傷つけない（感情の記録を過度に分析・評価しない）
- Soraらしさを壊していない（シンプルで静かなリスト）
- AI依存を強めすぎていない
- 既存機能を壊していない
