# ISSUE-053: 「あの日のこと」AI通知を実装する

name: ISSUE-053
type: feature
priority: High
status: done

---

## 概要

記録日から1ヶ月・3ヶ月・1年後にAIが記録を読んで生成した通知を送り、過去記録を能動的に浮上させる。

## 背景

記録は蓄積されているがユーザーが自分で見返す動機が弱い。「残してよかった」という体験は記録時ではなく「後から再会したとき」に生まれる。この体験を向こうから届ける仕組みがない。

## 目的

- 過去記録の価値を「後から再会」によって体験させる
- 記録を増やすほど将来の体験が豊かになる動機を作る
- ロスケアモード中の継続率を支える

## スコープ

- 記録保存時に1ヶ月後・3ヶ月後・1年後の通知をスケジュール登録
- 通知文生成
  - Claude APIに元の記録（タイトル・本文・気持ちタグ）を渡して生成
  - 形式：「1年前の今日、{ペット名}と〇〇でしたね。」（1〜2行）
  - ロスケアモード中は文体を変更（懐かしむ語り口・「残してくれてよかった」）
- 通知タップで該当記録画面に直接遷移
- プッシュ通知許可がないユーザーはスキップ（強制しない）
- ユーザー設定で「あの日通知」のオン/オフ可能（Settings）

## 非スコープ

- 通知タイミングのカスタマイズ（1ヶ月・3ヶ月・1年は固定）
- 通知頻度の上限設定（記録数が多いユーザーへの配慮はPhase 2）

## API変更

- `POST /api/pets/[petId]/memories` の保存処理に通知スケジュール登録を追加
- 通知スケジュール管理テーブル `memory_notifications` を追加
  - `memory_id`, `scheduled_at`, `type`（1m/3m/1y）, `sent_at`

## 受け入れ条件

- [ ] 記録保存時に3件の通知がスケジュールされる
- [ ] 指定タイミング（±1日以内）に通知が送信される
- [ ] 通知文が元の記録内容を反映している
- [ ] 通知タップで該当記録に遷移する
- [ ] Settings からオン/オフできる
- [ ] ロスケアモード中は文体が変わる
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: 記録保存→通知スケジュール登録確認
- 正常系: 通知タップ→記録画面遷移
- 正常系: Settings でオフ→通知が送られない
- 文体確認: 通常時とロスケア時で文体が異なる
