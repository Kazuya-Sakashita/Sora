# 概要

記録・写真・AIの手紙をもとに、PDFまたはデジタルブック形式の「メモリアルブック」を自動生成する機能を追加する。「あの子の本」として手元に残せる形にする。

# 背景

記録が蓄積されてきたユーザーにとって、それをひとつの「本」にまとめたいという欲求は自然に生まれる。スクリーンショットを並べることなく、Sora上で完結してブックを作れることが価値になる。Sora+の有力なマネタイズポイントでもある。

# 解決したい課題

- 記録が蓄積されても「見返す」体験がタイムライン閲覧に限られている
- 大切な記録を外部に保存・印刷したいニーズへの対応がない
- Sora+の差別化要素が現状弱い

# ユーザー価値

「あの子の記録をひとつの本にまとめられた」という完結感。手元に残る形での記憶の保存。

# 実装方針

**ブック生成フロー:**
- 対象ペットを選択 → 期間・写真枚数・含めるコンテンツを選択 → 生成
- 含められるコンテンツ：写真（選択式）・記録テキスト・気持ちタグ統計・AIの手紙1〜3通・ペットプロフィール

**出力形式（MVP）:**
- PDF（印刷・保存可能）
- A4縦、写真グリッド＋テキストの組み合わせレイアウト

**API変更:**
- `POST /api/pets/{petId}/memorial-book` — ブック生成ジョブ開始
- `GET /api/pets/{petId}/memorial-book/{jobId}` — 生成状況確認・ダウンロードURL取得

**UI:**
- 設定画面またはタイムライン画面に「メモリアルブックを作る」ボタン
- 生成中はローディング表示
- 完成後はダウンロードリンク（7日間有効）

**Sora+連携:**
- 生成はSora+限定機能（無料ユーザーはプレビューのみ）

# 受け入れ条件

- [x] 対象ペットのメモリアルブック生成が開始できる
- [x] 生成完了後にPDFをダウンロードできる
- [x] 写真・テキスト・AIの手紙がレイアウトされている
- [x] Sora+ユーザーのみ完全生成できる（無料はプレビュー表示まで）

# やらないこと

- 印刷物の発送（物理的な本の製造）
- 完全カスタムデザイン（テンプレート固定）
- リアルタイム編集（生成後の細かい調整）

# 検証方法

- メモリアルブック生成率（対象ペットあたり）
- 生成ユーザーのSora+転換率
- ダウンロード後のシェア率
