# ISSUE-040: 初回記録への強制誘導（First Record Nudge）

name: ISSUE-040
type: feature
priority: Critical
status: open

---

## 概要

プロフィール作成完了後、思い出が0件のユーザーに「最初の思い出を残す」CTAを1回だけ表示し、Activation を完結させる。

## 背景

現状はプロフィール作成 → ホーム遷移で終わる。最初の記録を促すフローがなく、「記録する」という行動習慣が形成される前に離脱するユーザーが多い。D1リテンションに直結する。

## 目的

- 初回セッション内で記録1件を完了させる（Activation 完了）
- 習慣化の起点を作る

## スコープ

- `home.tsx`：`memories.length === 0` かつ `pet` が存在する場合、ホーム最上部に専用バナーを表示
  - コピー例：「${pet.name}との最初の思い出を残しましょう」
  - ボタン：「今すぐ残す」→ タイムラインの記録フォームを開く
- 1件でも記録があれば二度と表示しない（`memories.length > 0` で非表示）
- バナーは GlassCard スタイルで統一

## 非スコープ

- ローカルストレージへの「表示済み」フラグ保存（memoryが0件かで判定するため不要）
- チュートリアル系の複数ステップUI

## API変更

- なし

## 受け入れ条件

- [ ] 記録0件のホームにバナーが表示される
- [ ] ボタンタップでタイムライン画面の記録フォームが開く
- [ ] 1件記録後にバナーが消える
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: memories.length === 0 でバナー表示
- 正常系: memories.length > 0 でバナー非表示
- UI動作: ボタンタップでタイムライン遷移
