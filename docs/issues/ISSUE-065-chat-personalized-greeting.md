# ISSUE-065: チャット初期メッセージをペット名でパーソナライズする

name: ISSUE-065
type: improvement
priority: High
status: done

---

## 背景

現在のチャット画面の初期メッセージは「ここにいるよ。無理しなくて大丈夫」という汎用的な文言で、ペット名も含まれていない。初めてチャットを開いたとき、あるいはリセット後に再開したとき、「Soraが自分のペットのことを知っている」という感覚が得られない。

## 問題

`INITIAL_MESSAGE` はコンポーネント外の定数で定義されており、pet情報を参照できない。すべてのユーザーが同じ初期文言を受け取る。

## 目的

初期メッセージをペット名を使ってパーソナライズし、「Soraはあなたのペットのことを知っている」という体験を最初の一言から届ける。ペットが設定されていない場合は現状のフォールバックを維持する。

## スコープ

- コンポーネント外の `INITIAL_MESSAGE` 定数を削除
- `ChatScreen` 内で `pet?.name` を使って初期メッセージを動的生成
  - ペット名あり: `${pet.name}のことを話せる場所です。ゆっくりでいいですよ。`
  - ペット名なし: `ここにいるよ。ゆっくりでいいですよ。`
- localStorage 復元時も reset 時も同じ関数で初期メッセージを生成
- `INITIAL_MESSAGE` を参照している箇所をすべて関数化

## やらないこと

- AIによる動的な初期メッセージ生成（API呼び出しなし）
- 直近記録を初期メッセージに含める

## 受け入れ条件

- [ ] チャット初回表示でペット名が初期メッセージに含まれる
- [ ] リセット後もペット名を含む初期メッセージが表示される
- [ ] localStorage復元時は保存されていたメッセージが表示される（ペット名入りの初期メッセージで上書きしない）
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] gstackで動作確認済み
