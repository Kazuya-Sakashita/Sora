# ISSUE-051: 月次AIメモリーレターをSora+機能として実装する

name: ISSUE-051
type: feature
priority: High
status: open

---

## 概要

毎月末にその月の記録をAIが読み返し、200〜300字の手紙形式でSora+ユーザーに届ける。

## 背景

現在の課金機能（PDF・家族共有）はすべて「作って終わり」のワンショット価値。毎月課金し続ける感情的な理由がない。月次レターは「使い続けるほど価値が深まる」サブスク体験の核になる。

## 目的

- Sora+の毎月の解約理由をなくす
- 「私とこの子の話を誰かが覚えていてくれる」体験を作る
- MRR維持率を改善する

## スコープ

- 毎月末日23:59にバッチ処理でSora+ユーザー全員分を生成
  - Claude APIにその月の記録全件（タイトル・本文・気持ちタグ・日付）を渡す
  - 200〜300字・手紙形式・ペット名と実際の記録内容を含む・汎用文禁止
- 生成完了後にプッシュ通知「{ペット名}との今月のレターが届きました」
- Settings → Sora+セクションに「月次レター」タブを追加
  - 月別アーカイブで過去レターを閲覧可能
- Freeユーザー：前月レターのサンプルを1回だけ表示してアップグレード誘導
- 記録が3件未満の月はレター生成をスキップ（翌月に持ち越さない）
- 生成失敗時は翌日リトライ1回。それでも失敗した場合はスキップ

## 非スコープ

- レターへの返信・コメント機能
- レターのPDF出力

## API変更

- `GET /api/pets/[petId]/letters` 月次レター一覧取得
- `GET /api/pets/[petId]/letters/[year]/[month]` 特定月のレター取得
- バッチ処理: `scripts/generate-monthly-letters.ts`（cron想定）

## 受け入れ条件

- [ ] 月末にSora+ユーザーのレターが自動生成される
- [ ] レター文にペット名と実際の記録内容が含まれる
- [ ] プッシュ通知でレター到着が通知される
- [ ] Settings画面から過去レターを月別で閲覧できる
- [ ] Freeユーザーにサンプル表示→アップグレード誘導が機能する
- [ ] 記録3件未満の月はスキップされる
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: レター生成・通知・閲覧の一連フロー
- 正常系: Freeユーザーのサンプル→アップグレード誘導
- 境界値: 記録2件の月はスキップされる
- 異常系: API失敗時のリトライとスキップ
