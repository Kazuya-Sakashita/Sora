# ISSUE-015: 記録完了後の達成フィードバックを実装する

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | medium |
| ステータス | open |
| 関連画面 | 記録入力画面（完了時） |
| 関連ファイル | `components/screens/memory-create.tsx`（または記録入力コンポーネント） |

## 問題の概要

記録を保存した後のフィードバックがなく、「残せた」という充足感が生まれない。記録習慣の形成には、行動直後のポジティブなフィードバックが不可欠。

## 詳細

仕様再考（`docs/specs/sora-product-rethinking.md`）において、「残せた」という達成感がMVPの継続設計の核心と定義された。

**行動後フィードバックの重要性:**
- 記録 → 達成感 → また記録したい、というループを作る
- 「保存しました」だけでは感情価値がゼロ
- Soraらしい温かいコピーで「記録する行為の意味」を肯定する

## 期待する動作

記録保存後に、画面上部または中央に短時間表示されるトースト / モーダル:

```
✨ 残せました
今日の思い出が、ずっとここにあります。
```

または、画面遷移時のアニメーション + コピー:

```
保存完了アニメーション（✓マーク + フェードイン）
「[ペット名]との今日が、残りました。」
```

**設計条件:**
- 表示時間: 2〜3秒
- 強制的でない（すぐに消える、タップで消せる）
- コピーは「管理的」でなく「感情的」

## 実装方針

1. 記録保存成功時にトーストまたは簡易モーダルを表示
2. 表示後にタイムライン or ホームへ遷移
3. アニメーションは Tailwind の `transition` / `animate` を使用

## 完了条件

- [ ] 記録保存後にフィードバックが表示される
- [ ] コピーが感情的・温かい内容になっている
- [ ] 2〜3秒後に自動で消えるか、タップで消せる
- [ ] コンソールにエラーが出ていない
- [ ] ステータスを done に更新した
