# ISSUE-092: 長期未記録後の「おかえり」カードをホームに実装する

name: ISSUE-092
type: improvement
priority: High
status: open

---

## 背景

記録を7日以上していないユーザーがホームを開いたとき、現在は通常のホーム画面がそのまま表示される。プッシュ通知のリマインダーはあるが、「戻ってきた」ユーザーを温かく迎える体験が設計されていない。

alive期ユーザーが一度離脱して久しぶりに戻ってきたとき、「また始めてみよう」という気持ちになれる静かな一言が必要。

## 問題

```
// ホームを7日ぶりに開いても、何も変わらない
// 「ずいぶん来ていなかったな」と感じる体験設計がない
```

## 目的

長期間記録が空いた後にホームを訪れたユーザーに、義務感ではなく「また話せる」という安心感を届ける。

## スコープ

- `home.tsx` に以下を追加:
  - 表示条件:
    - `memories.length > 0`（過去に記録あり）
    - 最後の記録日から7日以上経過している
    - 今日まだ記録していない
    - localStorage に `sora:comeback-dismissed-{petId}-{year}-{month}` がない
  - カードUI（GlassCard）:
    - 絵文字なし（重みを出す）
    - タイトル: 「{N}日ぶりですね」（Nは最後の記録からの日数）
    - 本文: 「ゆっくりでいいですよ。{pet.name}のこと、また話しましょう。」
    - ボタン: 「記録する」→ タイムライン遷移
    - dismissボタン（✕）: localStorage に保存して今月は非表示
  - 表示位置: ホームの最上部（Pushバナーの下、ストリーク表示より上）
  - 計算はフロントエンドのみ（APIコールなし）

## やらないこと

- rainbow_bridge期への適用（ロスケア期は別の設計）
- 初回ユーザー（records === 0）への表示
- 30日超え・90日超えなど段階的なメッセージ変化（MVP）

## 受け入れ条件

- [ ] 7日以上未記録のalive期ユーザーにカードが表示される
- [ ] 記録0件・today記録済み・7日以内の場合は表示されない
- [ ] ✕ボタンで今月は非表示になる
- [ ] 「記録する」でタイムラインに遷移する
- [ ] rainbow_bridge期には表示されない
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過

## 確認観点

- 「サボっていた」という責め方をしていないか（義務感でなく安心感）
- Soraらしい静かなトーンを保っているか
- 1日に1回だけ表示されれば十分（毎回表示は重い）
