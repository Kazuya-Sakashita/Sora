# ISSUE-086: 「はなす」のフレーミングをカウンセリングから思い出整理に修正する

name: ISSUE-086
type: improvement
priority: Critical
status: done

---

## 背景

現在の「はなす」機能は、設計意図は「思い出を穏やかに語る場所」だが、UIとシステムプロンプトの複数箇所がAIカウンセリング的な体験を作り出している。

急性悲嘆状態のユーザーが「寄り添ってくれるAI」として依存的に使い始めるリスクがある。tone selector の「少し前を向く言葉もほしい」は特に危険で、ユーザーがAIに「悲嘆の方向性を決めてもらう」構造になっている。

```tsx
// chat.tsx — toneがカウンセリング的な期待を作る
"やさしく寄り添う" | "思い出を一緒に振り返る" | "少し前を向く言葉もほしい"
//                                                 ↑ これが問題
```

## 問題

1. tone selector の「少し前を向く言葉もほしい」がAIに回復方向を委ねる構造
2. 初期メッセージ「{pet.name}のことを話せる場所です」は正しいが、機能名「はなす」単体だとカウンセリング連想が強い
3. システムプロンプトの「穏やかに次の一歩を感じさせる言葉を添える」指示が前向き誘導を許容している

## 目的

「はなす」を「思い出を整理する補助ツール」として明確に位置づける。AIがユーザーの感情方向を変えようとしない設計にする。

## スコープ

### 1. tone selector から「少し前を向く言葉もほしい」を削除
- `components/screens/chat.tsx` の quickTones または tone選択UIから当該オプションを削除
- Settings画面にも同オプションがあれば削除

### 2. 対応するシステムプロンプト分岐を削除
- `app/api/pets/[petId]/chat/route.ts` の toneInstruction:
  ```ts
  // 削除する分岐
  : tone === "少し前を向く言葉もほしい"
  ? "- 悲しみを受け止めつつ、ごく穏やかに次の一歩を感じさせる言葉を添える（押しつけ禁止）"
  ```
- 残す tone: `"やさしく寄り添う"` と `"思い出を一緒に振り返る"` のみ

### 3. チャット初期メッセージのフレーミング追加（optional）
- 現在: `{pet.name}のことを話せる場所です。ゆっくりでいいですよ。`
- 変更: そのまま維持（十分に正確）

## やらないこと

- チャット機能自体の廃止・制限
- システムプロンプト全体の書き直し
- tone選択UI自体の廃止
