# ISSUE-005: タイムラインの空状態設計が不足

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | medium |
| ステータス | done |
| 関連画面 | タイムライン画面 |
| 関連ファイル | `components/screens/timeline.tsx` |

## 問題の概要

記録が0件のときのタイムライン画面が冷たい・最初の記録追加への導線が不明確。

## 詳細

新規ユーザーが最初にタイムラインを開いたとき、空のリストが表示される状態は「このアプリ使えるのかな」という不安を生む。仕様書では「空状態が『冷たく』ない」ことをUX品質基準として定義している。

## 期待する動作

- 空状態に温かいメッセージと最初の記録への誘導ボタンがある
- 例: 「まだ思い出がありません。[名前]との最初の思い出を残しませんか？」
- 記録追加ボタンまたは記録画面へのリンク

## 完了条件

- [ ] 0件時に適切な空状態UIが表示される
- [ ] 記録追加への導線がある
- [ ] 既存機能に影響なし確認済み
- [ ] ステータスを done に更新した
