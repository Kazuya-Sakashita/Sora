# ISSUE-073: オンボーディングにAI機能の価値訴求を追加する

name: ISSUE-073
type: improvement
priority: High
status: open

---

## 背景

現在のオンボーディングはペットプロフィール登録 → 初回記録完了で終わる。ユーザーは「写真と日記のアプリ」として認識してアプリを使い始める。

Soraが他のペット日記アプリと決定的に異なるのは、記録がAIによるパーソナルな応答・月次レター・「あの日のこと」通知のコンテキストとして機能する点だ。この価値がオンボーディングで一切伝わっていないため、AI機能（はなす・ことば）への到達率が低く、差別化が伝わらない。

## 問題

オンボーディングフローで「記録がAIにつながる」体験予告が存在しない。ユーザーはAI機能の存在を知らないままアプリを使い続ける可能性がある。

## 目的

初回記録完了後（ISSUE-052で実装したフロー末尾）に1枚スライドを追加し、「記録が積み重なると、Soraが語りかけてくれます」という価値予告をする。記録へのモチベーションとAI機能への期待値を同時に植え付ける。

## スコープ

- 初回記録完了後の完了画面（`onboarding.tsx` または `first-record` 完了ステップ）に1枚追加
- 表示内容:
  - 絵文字: `✨`
  - タイトル: 「記録が、Soraとのつながりになります」
  - 説明文（2行以内）: 「記録が積み重なると、Soraがその子との日々を振り返り、あなたに語りかけます。」
  - ボタン: 「はじめる」→ ホーム画面に遷移
- 既存の完了ステップUIのデザイントーンに合わせる
- 表示タイミング: 初回記録保存後、ホーム遷移前の1回のみ（localStorage で表示済みフラグ管理不要 — オンボーディングフロー自体が1回限り）

## やらないこと

- オンボーディング全体のリデザイン
- AI機能のデモ・インタラクティブチュートリアル
- スキップ機能の追加

## 受け入れ条件

- [ ] 初回記録完了後にAI価値訴求スライドが表示される
- [ ] 「はじめる」ボタンでホーム画面に遷移する
- [ ] 既存オンボーディングフローの他ステップが壊れていない
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過

## 確認観点

- 過度な期待値（「AIが何でも解決する」）を植え付けない
- Soraらしい穏やかなトーン
- 既存機能を壊していない
