# 概要

現在のSora+（有料プラン）の価値提案が「AIの手紙が多く届く」だけでは弱い。ユーザーが「これのために払う」と思える価値に再設計する。

# 背景

製品評価（sora-product-reassessment.md）でSora+の収益化が最大の課題として特定された。現状の有料機能はAI手紙の受信上限緩和程度で、課金の動機として弱い。「無料でも十分使える → 課金理由がない」という構造を変える必要がある。

# 解決したい課題

- 無料版と有料版の体験差が薄い
- 「これは払う価値がある」と感じる機能がSora+にない
- 収益なしには開発・サーバーコストを支えられない

# ユーザー価値（Sora+再定義）

「大切な記録を完全に残し、深く向き合うための全機能」をSora+に集約する。

# 実装方針

**Sora+機能パッケージ（再設計案）:**

| 機能 | 無料 | Sora+ |
|------|------|-------|
| 記録件数 | 無制限 | 無制限 |
| 写真保存 | 1ペット50枚 | 無制限 |
| AIの手紙 | 月1通 | 月4通 |
| メモリアルブック | プレビューのみ | 生成・DL可能 |
| ひとこと記録 | 無制限 | 無制限 |
| 感情トレンド | 3ヶ月 | 全期間 |
| 公開プロフィール | なし | あり |
| あの子らしさVault | 3項目 | 全5項目 |
| 優先AIレスポンス | なし | あり |

**価格設計（現状から変更検討）:**
- 月額480円 → 月額680円（価値向上に伴い）
- 年払い：月換算450円（年5,400円）
- 初月無料トライアル（Rainbow Bridge期ユーザーは2ヶ月無料）

**Sora+ページ改善（`/sora-plus`）:**
- 現在の機能一覧に「これがないと何ができないか」を明記
- ユーザーの声（定性）を追加
- Rainbow Bridge期ユーザー向けの特別コピー

**API変更:**
- `GET /api/subscription` レスポンスに `planFeatures` オブジェクト追加
- 各機能の利用可否をフロントが判断できるようにする

# 受け入れ条件

- [ ] Sora+の機能テーブルが明確に定義・実装されている
- [ ] 無料ユーザーが制限に触れたとき、Sora+へのupsellが自然に表示される
- [ ] Rainbow Bridge期ユーザー向けの特別トライアルが機能する
- [ ] `/sora-plus` ページが新しい価値提案を反映している

# やらないこと

- 記録件数の無料制限（コアバリューを守る）
- AI手紙を完全有料化する（無料でも月1通は体験できる）
- 価格を1,000円超に設定する（日本のアプリ相場感）

# 検証方法

- 無料 → Sora+転換率（upsellトリガー別）
- Sora+解約率の変化
- Rainbow Bridge期ユーザーのSora+利用率
