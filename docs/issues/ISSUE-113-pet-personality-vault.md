# 概要

ペットの「らしさ」を細部まで残せる専用セクションをプロフィールに追加する。好きな場所・ご飯・癖・表情・嫌いなことを自由テキストで記録でき、忘れてしまいそうな記憶を形にして残せるようにする。

# 背景

JTBDの分析で「忘れてしまうことへの恐怖を手放したい」が最も切実なJobとして特定された。写真と日付だけでは「あの子らしさ」を残せない。「散歩に行くと必ず電柱で止まった」「ご飯の前に必ずくるくる回った」という細部が、最もその子を思い出させる記憶なのに、記録できる場所がない。

# 解決したい課題

- 「あの子の癖を忘れてしまいそう」という恐怖
- プロフィール情報が名前・種類・写真しかない
- AIが「あの子らしさ」を知らずに汎用的な反応を返している

# ユーザー価値

「ここに書いておけば忘れない」という安心感。また、AIが「あの子らしさ」を踏まえた反応・手紙・会話をできるようになる。

# 実装方針

**プロフィール編集画面への追加（settings.tsx + API）:**

新セクション「{name}のこと」:
- 好きな場所（自由テキスト、100文字）: placeholder「よく行った公園、ソファの隅、日当たりのいい窓辺…」
- 好きだったもの（自由テキスト、100文字）: placeholder「おやつ、おもちゃ、誰の膝が好きだったか…」
- よくやっていた癖・行動（自由テキスト、200文字）: placeholder「帰ってくると必ず…、眠るときいつも…」
- いちばん好きな顔・表情（自由テキスト、100文字）: placeholder「嬉しいときの顔、眠そうな顔…」
- 一緒に過ごしていた日常のこと（自由テキスト、200文字）: placeholder「朝のこと、夜のこと、決まってやっていたこと…」

**API変更:**
- Pet モデルに `personalityVault: String?` を追加（JSON文字列またはカラム追加）
- `PATCH /api/pets/{petId}` で更新
- `GET /api/pets/{petId}` のレスポンスに含める

**AI連携:**
- `personalityVault` の内容をAI反応・チャット・手紙生成のコンテキストとして注入
- 例：「ご飯の前に必ずくるくる回っていた」という情報があれば、手紙に反映できる

**公開プロフィールへの表示（任意）:**
- `/p/{petId}` の公開ページに「{name}のこと」セクションを表示（ユーザーが公開したい場合のみ）

# 受け入れ条件

- [ ] ペットプロフィール編集に「あの子のこと」セクションが追加されている
- [ ] 5項目のフリーテキスト入力が保存できる
- [ ] 保存した内容がAIチャット・手紙生成のプロンプトに渡されている
- [ ] 公開プロフィールで任意表示できる
- [ ] 入力は完全任意（未記入でもペット登録は完結する）

# やらないこと

- 必須入力にする
- 選択式（ドロップダウン・チェックボックス）にする（自由テキストの良さを残す）
- 医療・投薬情報の入力（健康記録の文脈に引き込まない）

# 検証方法

- 記入率（設定画面を開いたユーザーのうち何%が記入したか）
- 記入後の手紙・チャット品質の定性評価
- 記入したユーザーの長期Retention比較
