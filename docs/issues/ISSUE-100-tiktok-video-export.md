# ISSUE-100: 記録スライドショーの縦型動画（TikTok/Reels対応）を出力する

name: ISSUE-100
type: feature
priority: High
status: open

---

## 概要

直近の記録写真からアニメーション付き縦型スライドショー動画（9:16・15秒）を生成し、TikTok・Instagram Reels・LINEへのシェアを可能にする。

## 背景

Instagramストーリーへの静止画シェア（ISSUE-098）に加え、動画フォーマットはTikTok・Reelsで拡散力が高い。ペット動画は最も高エンゲージメントなカテゴリの1つ。

ユーザーが手間なく「うちの子動画」を作れると、TikTokへの投稿動機が生まれ、Growth Loop 1が加速する。

## 目的

TikTok・Reels・LINEに投稿できる高品質な縦型ペット動画を0コストで作れるようにし、SNS経由の新規ユーザー流入を生み出す。

## スコープ

- タイムライン画面または月次カード画面から「動画を作る」ボタンを追加
- 直近5〜10件の記録から写真を選択（自動または手動）
- Canvas API + requestAnimationFrame でスライドショーアニメーション生成
  - フェードイン・ケン・バーンズ効果
  - 各写真に記録日付とタイトルをテロップ表示
  - BGMなし（著作権問題を避けるため）
- MediaRecorder API で mp4 / webm 録画 → ダウンロード
- 生成中はローディング表示
- 写真がない記録はスキップ

## 非スコープ

- BGM追加機能
- サーバーサイドでの動画変換（クライアントサイドのみ）
- 60秒超の長尺動画

## API変更

なし

## 受け入れ条件

- [ ] 写真付き記録が3件以上あれば動画生成ボタンが表示される
- [ ] 縦型（9:16）の動画が生成される
- [ ] 動画をダウンロードできる
- [ ] Web Share API 経由でシェアできる（対応ブラウザ）
- [ ] 写真が1枚もない場合はボタンが非表示

## テスト観点

- 正常系: 写真3件以上で動画生成
- 境界値: 写真0件でボタン非表示、1〜2件の場合の挙動
- UI動作: 生成中ローディング、完了後のダウンロードボタン
