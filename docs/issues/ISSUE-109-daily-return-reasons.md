# 概要

「今日記録することがあれば来る」という受動的なアプリから、「今日来なければ見逃す何かがある」能動的なアプリに変える。1年前の今日・月次の写真再サーフェシング・記念日の自動検出・週次の気持ちサマリーを組み合わせてホームに表示する。

# 背景

現在のRetention阻害の主因は「来る理由が記録衝動だけ」という設計にある。記録したいことがない日は来なくていい、というアプリになっている。しかし「今日あの子のことを思った」だけで来ていい理由を設計する必要がある。

# 解決したい課題

- 記録したいことがない日にアプリを開く動機がない
- 「1年前の今日」が通知だけで、ホーム画面に来ない
- 記念日（お迎え日・誕生日）の特別扱いが弱い
- 気持ちの変化が「過去ふり返り」の形で届かない

# ユーザー価値

「今日も来てよかった」という毎日の小さな充足感。習慣が生まれる。

# 実装方針

**①「1年前の今日」カードのホーム内表示（home.tsx）:**
- 現在は通知のみ → 通知に加えてホームカードとしても表示
- カード内容：1年前の記録のタイトル + 写真（あれば） + 「あの頃の{name}」
- タップ → タイムラインの該当記録にスクロール

**②月初の「先月の{name}」写真カード:**
- 毎月1日のホームに、先月投稿された写真からランダム1枚を大きく表示
- 「先月の{name}」というシンプルなキャプション
- 写真がない月はスキップ

**③記念日の自動検出と特別表示:**
- broughtAt（お迎え日）の年次記念日：「{name}を迎えて{n}年になりました」
- birthDate（誕生日）：「今日は{name}の誕生日です」
- これをホームカードとして表示（milestone cardとは別）
- Push通知も送る（on-this-dayと同様のシステムを活用）

**④「先週の気持ち」週次サマリー（毎週月曜）:**
- 先週の気持ちタグ集計：「先週いちばん多かった気持ちは{tag}でした」
- 一言のみ（グラフなし・テキストのみ）
- Alive期：「今週も一緒に記録しましょう」
- RB期：「今週は{name}のことを何回思いましたか」

# 受け入れ条件

- [ ] ホーム画面に「1年前の今日」カードが（通知とは別に）表示される
- [ ] 毎月1日に先月の写真カードが表示される
- [ ] broughtAt / birthDate の年次記念日にホームカードとPush通知が届く
- [ ] 毎週月曜に先週の気持ちサマリーが1行で表示される
- [ ] これらのカードは1日1件まで（重なった場合は優先度順に1件）

# やらないこと

- すべての記念日を毎年Push通知する（通知疲れ → broughtAt/birthDate のみ）
- 気持ちの週次サマリーにグラフや統計詳細を出す（1行テキストで十分）
- 「今日も記録しましょう」という催促的コピー

# 検証方法

- 各カードの開封率（タップ率）
- カード表示日 vs 非表示日のセッション時間比較
- 月次Push通知（記念日）のクリック率
