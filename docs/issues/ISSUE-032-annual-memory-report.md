# ISSUE-032: 年次メモリーレポートを自動生成する（Sora+）

name: ISSUE-032
type: feature
priority: Critical
status: open

---

## 概要

Sora+ ユーザーの1年分の思い出を自動で PDF レポートとして生成し、毎年1月1日に通知で届ける。

## 背景

「記録し続けた先に何があるか」が見えないと継続・課金動機にならない。年次レポートは Sora+ の看板価値であり、「これのために払う」と言わせる機能。

## 目的

Sora+ の解約抑止と新規転換の両方に効く「1年分の思い出が本になる」体験を提供する。

## スコープ

- `@react-pdf/renderer` をインストールし年次レポート PDF テンプレートを実装
  - 表紙（ペット名・年・ペット写真）
  - 月別サマリー（記録件数・写真枚数・最多気持ちタグ）
  - 記念日ハイライト（100日・誕生日・お迎え記念日）
  - 思い出ピックアップ（写真付き上位5件）
- `GET /api/pets/:petId/report?year=YYYY` — PDF 生成・返却（Plus 限定、`requirePlus` チェック）
- タイムライン画面に「{year}年の思い出レポートを見る」ボタン追加（Plus のみ表示）
- Vercel Cron `0 0 1 1 *`（毎年1月1日 UTC 0時）でプッシュ通知送信

## 非スコープ

- 印刷所連携・物理製本
- テンプレートの複数選択
- 月次・週次レポート（年次のみ）

## 受け入れ条件

- [ ] PDF が正しく生成・ダウンロードできる（写真・月別サマリー・記念日含む）
- [ ] Free ユーザーはボタンが Upgrade モーダルに繋がる
- [ ] 記録が0件の年は「まだ記録がありません」PDF を返さず 404
- [ ] Cron が1月1日に Plus ユーザーへ通知を送る
