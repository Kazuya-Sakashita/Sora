# ISSUE-006: アプリタイトルが「ペットロスヒーリング」でロス後専用に見える

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | critical |
| ステータス | open |
| 関連画面 | 全画面（ブラウザタブ・シェア時） |
| 関連ファイル | `app/layout.tsx` |

## 問題の概要

`metadata.title` が `'にじのはし - ペットロスヒーリング'` になっており、生前から使うアプリとして全く機能しない。

## 詳細

```tsx
// app/layout.tsx（現状）
export const metadata: Metadata = {
  title: 'にじのはし - ペットロスヒーリング',
  description: '大切な思い出と、ゆっくり向き合うための場所です',
}
```

ブラウザのタブ・ブックマーク・OGシェア時にこのタイトルが表示される。「ペットロスヒーリング」という言葉を見た時点で「亡くなったペットのためのアプリ」と認識され、生きているペットの飼い主はインストールしない。仕様書の最重要方針「生前から自然に使える・ペットロス専用に見せない」を根本から壊している。

## 期待する動作

- アプリ名: `Sora`（または仮称のまま開発継続）
- タイトル: `Sora - ペットとの毎日を残す場所`
- description: `写真や日々の記録が、ずっと大切な思い出になる`
- アプリ名「にじのはし」は仮称。最終決定まで `Sora` で統一する

## 実装方針

`app/layout.tsx` の metadata を修正するだけ。

- [ ] タイトル・descriptionが生前利用を前提とした文言になった
