# ISSUE-064: チャット画面のalive期ガードを追加する

name: ISSUE-064
type: improvement
priority: Critical
status: done

---

## 背景

chat.tsx には pet.status のチェックがない。alive状態のペットオーナーが直接画面に遷移するとチャットUIが表示され、送信すると API 403 が返り「少し待ってから、もう一度試してみてください」という誤解を招くエラーが表示される。「はなす」はグリーフケア専用機能であることが伝わらず、エラーで終わる体験になっている。

## 問題

`ChatScreen` コンポーネントは `pet?.status !== "rainbow_bridge"` の場合にも完全なチャットUIを描画する。ユーザーがメッセージを送ると API が 403 を返すが、エラー文言はレート制限と同じ汎用文言で意図が伝わらない。

## 目的

alive期ユーザーがチャット画面に到達した場合、「この機能はロスケアモードで使えます」という穏やかなメッセージと記録画面への誘導を表示する。チャットUIは表示しない。

## スコープ

- `ChatScreen` の先頭で `pet?.status !== "rainbow_bridge"` を判定
- alive 状態ならチャットUIの代わりに「まだここではなせる日が来ていない」旨のカードを表示
- ホームへ戻るボタンを添える
- ペットがまだ設定されていない場合も同じガードで保護

## やらないこと

- ナビゲーション層での制御（app-contextやhome.tsxの変更）
- チャット機能の alive 対応（スコープ外）

## 受け入れ条件

- [ ] alive状態のペットでchat画面に遷移するとガードメッセージが表示される
- [ ] ガードメッセージからホームへ戻れる
- [ ] rainbow_bridge状態では通常のチャットUIが表示される
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] gstackで動作確認済み
