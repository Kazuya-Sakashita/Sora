# ISSUE-031: 記録上限ゲートで Free→Plus 転換を促す

name: ISSUE-031
type: feature
priority: Critical
status: open

---

## 概要

Free プランの記録を50件に制限し、上限接近・到達時に Sora+ へ誘導するゲートを実装する。

## 背景

全機能が無制限無料では課金動機が生まれない。記録が増えてきた「愛着のピーク」で自然に転換を促すことが最も転換率が高い。ISSUE-030 完了後に実装する。

## 目的

Free ユーザーの Sora+ 転換率を上げる。「大切な記録を続けるには」という文脈で課金を自然に誘導する。

## スコープ

- `POST /api/pets/:petId/memories` に件数チェック追加：`plan=FREE` かつ件数 ≥ 50 なら `402 Payment Required` を返す
- フロント: 49件目の記録保存成功後に「あと1件で上限です。Sora+ で無制限に残せます」バナーを表示（3秒）
- フロント: 50件到達時の保存失敗で Upgrade モーダルを表示
- Upgrade モーダル: プラン比較・「アップグレードする」→ `/api/billing/checkout` へ遷移

## 非スコープ

- 写真枚数制限（初回は件数のみ）
- feelings / schedules の件数制限

## 受け入れ条件

- [ ] Free ユーザーの50件目保存が API レベルで 402 を返す
- [ ] 49件目保存後にバナーが表示される
- [ ] 50件到達時に Upgrade モーダルが表示される
- [ ] Plus ユーザーは制限なく保存できる
- [ ] テスト: `plan=FREE` で50件超えた際の API 挙動
