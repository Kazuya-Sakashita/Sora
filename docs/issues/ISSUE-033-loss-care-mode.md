# ISSUE-033: ロスケアモード（Phase 3）をステータス変更で解放する

name: ISSUE-033
type: feature
priority: High
status: open

---

## 概要

ペットステータスを `rainbow_bridge` に変更した瞬間、手紙・追悼タイムラインなどの Phase 3 機能を自動解放する。

## 背景

ペットロス直後が最も感情的・課金意欲が高い瞬間。現状その機能が封印されており最大の機会損失。ロスケア機能は有料制限せず感情的障壁を作らない方針とする（Sora+ の信頼感を高める）。

## 目的

ロス後ユーザーに最も必要な機能を届け、「Soraに救われた」という体験を作る。長期的に口コミ・継続課金に繋げる。

## スコープ

- ペット設定画面に「虹の橋へ旅立った」ステータス変更フローを追加
  - 確認モーダル（穏やかな文言・取り消し可能）
  - `PATCH /api/pets/:petId` で `status: RAINBOW_BRIDGE` を更新
- `status === RAINBOW_BRIDGE` 条件で以下を表示解放：
  - 手紙画面（現在 Phase 3 フラグで非表示）
  - チャット画面（現在 Phase 3 フラグで非表示）
- ホーム画面: ステータスに応じてコピー切り替え
  - ALIVE: 「{name}と一緒に {N}日」
  - RAINBOW_BRIDGE: 「{name}と過ごした {N}日間」
- ナビゲーションに手紙・チャットタブを追加（RAINBOW_BRIDGE 時のみ）

## 非スコープ

- AI 生成の手紙文（倫理審査が必要）
- ロスケアコンテンツの有料化

## 受け入れ条件

- [ ] ステータス変更後に手紙・チャット画面がナビに表示される
- [ ] 変更前は非表示のまま
- [ ] 確認モーダルの文言に感情的配慮がある
- [ ] ステータスは ALIVE に戻せる（ペット設定から）
- [ ] ホームのコピーがステータスに応じて切り替わる
