# 概要

現在のリスト形式・カレンダー形式に加え、写真だけを大きく並べた「グリッドビュー」をタイムラインに追加する。写真を見返す行為それ自体が体験になるようにする。

# 背景

ペットとの記録において写真は主役であるはずだが、現在のリストビューではタイトル（テキスト）が情報の主体で、写真はサムネイルに過ぎない。「あの頃の写真を見返す」という行為のために最適化されたビューがない。

# 解決したい課題

- 「写真を眺める」体験への最適化がない
- 写真が少ない記録でもリストに埋もれて見えない
- 「あの頃どんな顔してたっけ」を確認する動線が遠い

# ユーザー価値

Soraが「アルバム」として機能する。写真を見返す行為が「また来た理由」になる。

# 実装方針

**タイムライン3つ目のビュー（timeline.tsx）:**
- 現在：List | Calendar の2ビュー
- 追加：List | Photo | Calendar の3ビュー
- Photoビューはデフォルトではなく、タブで切り替え

**グリッドレイアウト:**
- 3列グリッド（Instagramライク）
- 写真がある記録のみ表示
- 写真がない記録はグリッドから除外（写真のない日は表示しない）
- 月毎のセクションヘッダー：「2026年5月（{n}枚）」

**タップ時の詳細表示:**
- フルスクリーン写真 + 下部に記録タイトル + 日付 + 気持ちタグ
- 左右スワイプで前後の写真へ移動（写真の時系列ナビゲーション）
- 「この記録を見る」ボタンでリストビューの該当記録へ

**Rainbow Bridge期の特別演出:**
- グリッドの最上部に「{name}との思い出」というヘッダー
- セピアフィルターは使わない（写真本来の色を大切にする）

# 受け入れ条件

- [ ] タイムラインに「写真」ビュータブが追加されている
- [ ] 写真がある記録のみ3列グリッドで表示される
- [ ] 月毎のセクションヘッダーに写真枚数が表示される
- [ ] 写真タップでフルスクリーン表示 + 基本情報が見える
- [ ] フルスクリーン時に左右スワイプで前後の写真へ移動できる

# やらないこと

- 写真のフィルター加工・エフェクト
- 写真の自動補正・明度調整
- 写真がない記録のグリッド内表示（プレースホルダーも置かない）

# 検証方法

- 写真グリッドビューの利用率（ビュー切り替え率）
- グリッドビュー利用後のセッション時間
- フルスクリーン表示の写真滞在時間
