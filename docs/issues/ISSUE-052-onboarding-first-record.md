# ISSUE-052: オンボーディング最後に初回記録を必須完走させる

name: ISSUE-052
type: feature
priority: High
status: open

---

## 概要

オンボーディングの最終ステップに「最初の1枚を残す」を追加し、インストール当日に記録習慣の起点を作る。

## 背景

現在のオンボーディングはペット登録で終わる。初日に記録しないユーザーの7日継続率は業界標準で極めて低く、どれだけ後続施策を打っても入口が壊れていれば無意味。

## 目的

- オンボーディング完了＝最初の記録が存在する状態にする
- 「記録する」という行動の初回体験を最初に作る

## スコープ

- オンボーディング最終ステップに「{ペット名}との最初の記録を残しましょう」画面を追加
- 入力フォーム（最小限）
  - 写真: カメラ起動 or ライブラリ選択（任意）
  - タイトル: 必須（プレースホルダー「今日の{ペット名}」）
  - 気持ちタグ: 1つ選択（任意）
- スキップボタンは置かない
- 保存後にAIリアクション（ISSUE-049）が初回として表示される
- 既存ユーザー（ペット登録済み）にはオンボーディング再表示しない

## 非スコープ

- 本文（description）の入力（最初の記録は最小コストで）
- 複数写真アップロード

## API変更

- なし（既存の `POST /api/pets/[petId]/memories` を使用）

## 受け入れ条件

- [ ] オンボーディング完了後に記録が1件存在する
- [ ] タイトルなしでは保存できない
- [ ] 写真なし（テキストのみ）でも完了できる
- [ ] 既存ユーザーのフローに影響しない
- [ ] ISSUE-049のAIリアクションが保存後に表示される
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: 写真あり・タイトルありで完了
- 正常系: 写真なし・タイトルありで完了
- 異常系: タイトルなしで保存しようとするとエラー表示
- 既存ユーザー: オンボーディングが再表示されない
