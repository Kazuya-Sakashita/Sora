# ISSUE-010: チャット画面をPhase 3まで非表示化する

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | medium |
| ステータス | open |
| 関連画面 | お話しする（チャット）画面 |
| 関連ファイル | `components/screens/chat.tsx`, `app/page.tsx` |

## 問題の概要

チャット画面のクイックプロンプトと初期メッセージがロス後ユーザー専用で、生きているペットとの日常を語るユーザーには不自然。

## 詳細

```tsx
// 現状（問題）
const quickPrompts = [
  "会いたいな",           // ロス後
  "今日は少しつらい",     // ロス後
  "思い出を話したい",     // ロス後
  "ありがとうを伝えたい", // ロス後
]
// 初期メッセージ: 「ここにいるよ。無理しなくて大丈夫」← ロス後向け
```

## 仕様変更によるスコープ更新（2026-04-28）

MVP再定義（`docs/specs/sora-product-rethinking.md`）において、AI会話機能は **Phase 3（Month 7〜）** に移動した。

**理由:**
- 記録が蓄積される前にAI会話を提供しても「あなたとペットの話」ができない
- 感情的に不安全な体験が先に出るリスクが高い
- ロス後7日以内はAI会話をデフォルトOFFにする設計が必要で、MVPには含まない

**MVPでの対応方針（スコープ縮小）:**
1. チャット画面をナビゲーション・ボトムタブから非表示にする
2. コンポーネントファイル自体は残す（Phase 3で全面再設計して使用）

**Phase 3での再設計方針（参考）:**
- クイックプロンプト: 生前・ロス後どちらにも対応したものに変更
- 初期メッセージ: 「こんにちは。今日どんな一日でしたか？」等に変更
- ユーザーの記録データを文脈として読んだ上での対話設計

## 実装方針

`app/page.tsx` のルーティングからチャット画面を除外する。コンポーネント自体は削除しない。
