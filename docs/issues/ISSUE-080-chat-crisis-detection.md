# ISSUE-080: チャットAPIに危機フレーズ検出と専門サポート誘導を実装する

name: ISSUE-080
type: bug
priority: Critical
status: done

---

## 背景

`app/api/pets/[petId]/chat/route.ts` には、ユーザーが自傷・自殺念慮を示すフレーズを入力した場合の検出ロジックが存在しない。rainbow_bridge期のユーザーは急性悲嘆状態にあり、グリーフが深刻化した際に「死にたい」「消えてしまいたい」等をチャットに書く可能性がある。

現在の実装では、これらのフレーズが含まれていても Claude API を呼び出し、通常の grief care 返答が生成される。危機状態のユーザーに専門的なサポート情報が届かない。

## 問題

```ts
// chat/route.ts — どんなメッセージにも Claude API を呼ぶ
const reply = await generateChat(systemPrompt, messages.slice(-20))
return NextResponse.json({ reply })
```

自傷・危機フレーズへの早期リターンがない。

## 目的

チャットに危機フレーズが含まれる場合、Claude APIを呼ばずに専門サポートへ誘導する固定メッセージを返す。AIが「危機カウンセリング」をするのではなく、正しい出口に速やかに案内する。

## スコープ

- `app/api/pets/[petId]/chat/route.ts` に危機キーワードリストを実装
- 最後のユーザー発話（`messages[messages.length - 1].content`）にキーワードが含まれる場合:
  - Claude API を呼ばず早期リターン
  - 固定の安全誘導メッセージを返す:
    ```
    今、とても辛い気持ちなんですね。一人で抱えないでください。
    よりそいホットライン（0120-279-338）は24時間つながります。
    ```
- 検出キーワード（最小リスト、拡張可能な形で定義）:
  - 「死にたい」「死にたくなった」
  - 「消えてしまいたい」「消えたい」
  - 「もう生きていたくない」「生きていたくない」
  - 「自分を傷つけたい」「傷つけたい」
- キーワードリストは定数として別で定義（テスト可能・拡張可能）
- 早期リターン時もレート制限チェックは通過する（200を返す）

## やらないこと

- Claude API を使った危機対応プロンプトの実装
- フロントエンド側でのキーワード検出
- キーワードリストの自動更新
- 過去メッセージの遡及検出

## 受け入れ条件

- [ ] 「死にたい」を含む発話に対してよりそいホットラインを案内するメッセージが返る
- [ ] 通常の発話は検出に影響されない（false positive なし）
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過（危機検出のユニットテスト追加）
- [ ] 既存チャット機能を壊していない

## 確認観点

- ユーザーの心を傷つけない（最重要）
- Soraらしさを壊していない（穏やかな誘導文言）
- AI依存を強めすぎていない（人間の専門家へのブリッジ）
- 既存機能を壊していない
