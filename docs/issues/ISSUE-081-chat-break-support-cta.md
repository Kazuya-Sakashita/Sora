# ISSUE-081: チャット休憩メッセージに専門サポートへのCTAを添える

name: ISSUE-081
type: improvement
priority: Critical
status: done

---

## 背景

ISSUE-076でチャット8往復後の休憩提案メッセージを実装した。しかし現在の休憩メッセージは以下のテキストのみ:

```
今日はたくさん話してくれましたね。少し休んで、また話しましょう。
```

ユーザーが最も感情的に消耗したタイミングで、専門家への橋渡しが設計されていない。HeartHandshakeボタンはヘッダーに存在するが、休憩メッセージとは連動していない。

## 問題

```tsx
// chat.tsx — 休憩メッセージにサポートリンクがない
content: "今日はたくさん話してくれましたね。少し休んで、また話しましょう。"
// ↑ テキストのみ。専門サポートへのCTAなし
```

8往復後は最もサポートリンクが必要なタイミングだが、ユーザーはヘッダーのHeartHandshakeボタンの存在に気づいていない可能性が高い。

## 目的

休憩提案の直後に、HeartHandshakeサポートシートを開くインラインリンクを添える。「休んで → 専門家という選択肢もある」という流れを自然に作る。

## スコープ

- `chat.tsx` のbreak messageをテキストのみから「テキスト + サポートリンクボタン」に変更
- break messageのcontentはそのまま維持
- contentの下に小さなリンクボタンを追加:
  - テキスト: 「専門家に話してみる」
  - クリック: `setShowSupportSheet(true)` を呼ぶ
- break messageは通常のassistantバブルで表示するが、リンクボタン部分をUIとして区別するため、メッセージidを使ってrenderingで分岐する
  - Messages配列に `isBreakMessage?: boolean` フラグをMessage型に追加
  - break messageのみリンクボタンを表示するUI

## やらないこと

- チャット機能の変更
- HeartHandshake support sheetの内容変更
- 8往復の閾値変更

## 受け入れ条件

- [ ] 8往復後の休憩メッセージの下に「専門家に話してみる」リンクが表示される
- [ ] タップするとHeartHandshakeサポートシートが開く
- [ ] 通常のAIメッセージにはリンクが表示されない
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] 既存チャット機能を壊していない

## 確認観点

- ユーザーの心を傷つけない
- Soraらしさを壊していない（押しつけでない小さなリンク）
- AI依存を強めすぎていない（専門家への橋渡しを強調）
- 既存機能を壊していない
