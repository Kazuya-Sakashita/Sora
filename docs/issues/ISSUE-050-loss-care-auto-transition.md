# ISSUE-050: ロスケアへの自動移行フローを実装する

name: ISSUE-050
type: feature
priority: Critical
status: open

---

## 概要

ペットが亡くなった際、設定画面を操作させずに記録の流れの中で自然にロスケアモードへ移行する。

## 背景

現状はSettings画面でステータスを手動変更する設計（ISSUE-033で実装済み）。ペットを亡くした当日にアプリの設定を変える人間はいない。最も使ってほしいタイミングでの離脱がコアコンセプトを破壊している。

## 目的

- ロスケア移行の摩擦をゼロにする
- 「アプリが察してくれた」という安心感を作る
- ロスケアモード移行後の30日継続率を生前ユーザーと同水準にする

## スコープ

- ペットプロフィール編集画面に「虹の橋を渡りました」を追加
  - 既存のステータス変更UIと統合（Settings経由も残す）
- 選択後、確認画面を1枚表示
  - コピー例：「{ペット名}のことを教えてくれてありがとう。これからも、ここに残していいですよ。」
  - ボタン：「はい、移行します」「まだ大丈夫です」
- 確認後の変更内容
  - ホーム画面：「記録する」→「思い出を見る」へコピー変更
  - 記録追加ボタン：残す（ロス後も記録できる）
  - AIデイリー問いかけ（ISSUE-048）：文体を回想モードに切り替え
  - タイムライン：ヘッダーコピーを変更
- 移行後もロスケア解除・再設定が可能（Settings経由）

## 非スコープ

- ロスケアモード専用の特別コンテンツ（Phase 3）
- SNSへの追悼シェア機能

## API変更

- `PATCH /api/pets/[petId]` に `status: "passed"` を追加（既存APIの拡張）

## 受け入れ条件

- [ ] プロフィール編集から「虹の橋を渡りました」を選択できる
- [ ] 確認画面が表示され、キャンセルで戻れる
- [ ] 確認後にホーム・タイムラインのコピーが切り替わる
- [ ] ロス後も記録の追加投稿ができる
- [ ] Settings経由の手動変更も引き続き機能する
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: プロフィール編集→確認→移行の一連フロー
- 正常系: キャンセルで元の状態に戻る
- 正常系: 移行後も記録投稿ができる
- UI確認: ホーム・タイムラインのコピーが適切に切り替わっている
