# 概要

写真なし・タイトルなし・タグなしで「今日あの子のことを思った」を50文字以内で残せる「ひとこと記録」を追加する。記録の心理的ハードルを下げ、記録頻度を増やす。

# 背景

現在の記録単位は「タイトル（必須）+ 写真（任意）+ 気持ちタグ（任意）+ 説明（任意）」。タイトル入力でさえ「何を書けばいいか」と止まることがある。特にRainbow Bridge期は「記録したいことがあるわけじゃないけど、思い出した」という日が多い。

「今日ポチのことを思い出した」という事実だけでも残せる最小単位が必要。

# 解決したい課題

- 「タイトルが思いつかない」という心理的摩擦
- 「今日は記録するようなことがなかった」という日の記録欠落
- Rainbow Bridge期に「思い出した」だけでも残したい

# ユーザー価値

「記録する気力はないけど、今日{name}のことを思った」を残せる。高頻度・低摩擦。記録の蓄積が早まり、AI反応・手紙生成の材料が豊かになる。

# 実装方針

**UIエントリー（home.tsx）:**
- QuickRecordSheetの代替ではなく補完として、ホームに「今日のひとこと」入力行を追加
- テキストエリア（1行、placeholder: 「{name}のことをひとこと…」）
- Enterキー or 送信ボタンで即座に保存
- Rainbow Bridge期のホームでは、このエントリーを目立つ位置に置く

**記録タイプ（API・DB）:**
- Memory モデルに `category: "note"` を追加（または既存categoryを活用）
- タイトルは自動生成（日付＋「ひとこと」= 「5月10日のひとこと」）
- photoUrls = []、moodTag = null で保存

**タイムライン表示:**
- 「ひとこと」は通常の記録と区別したコンパクトなデザインで表示
- 写真なし・タグなし・1行テキストのみ
- 削除・編集可能

**AI反応:**
- ひとこと記録にも反応を返す（短く・優しく・1文のみ）

# 受け入れ条件

- [ ] ホーム画面から1タップ＋テキスト入力で「ひとこと」が保存できる
- [ ] 保存後にAIが1文で応答する
- [ ] タイムラインに「ひとこと」として通常記録と区別して表示される
- [ ] 通常の記録フローには影響しない
- [ ] Rainbow Bridge期のホームでこのエントリーが見つけやすい位置にある

# やらないこと

- ひとことを「正式な記録」と同等に扱う（別カテゴリとして明示）
- 字数制限を100文字超に広げる（長くなると記録と同じになる）
- 「今日も記録しましょう」というプッシュ的CTA文言

# 検証方法

- ひとこと記録の1日あたり件数
- ひとこと記録を使ったユーザーの週次Retention vs 未使用
- ひとこと記録 → 通常記録への転換率（「ひとことから始まった」記録）
