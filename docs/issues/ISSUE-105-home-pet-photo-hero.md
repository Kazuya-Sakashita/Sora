title: ホーム画面の「顔」をペットの写真に変える
type: ux
priority: critical
status: done
---

# 概要

ホーム画面を開いた瞬間、ペットの写真が大きく迎えてくれるようにする。現在はDaysカウンターとボタンが「顔」になっており、「管理ツールの画面」に見える。「あの子の場所に来た」という感覚を作る。

# 背景

テストユーザーから「パッとしない」「心が動かない」というフィードバックがあった。根本原因のひとつは、ホーム画面にペットの存在感がないこと。写真は小さなアバターに押し込まれており、名前は一行のテキストに過ぎない。

# 解決したい課題

- アプリを開いた瞬間「どこのアプリか」がわからない
- 「あの子の場所」という感覚がない
- 写真のないユーザーは最初から殺風景な印象を受ける

# ユーザー価値

アプリを開くたびに「ここに来てよかった」という感覚。特にロス後のユーザーにとって「あの子がいる場所」という印象は継続利用の根拠になる。

# 実装方針

- ヘッダー下にペット写真フルワイド表示（aspect-ratio: 4/3）
- 写真の下部に名前とひとこと（「{name}といっしょに」など）
- 写真なし時の代替：名前入りのグラデーション背景（空のイメージ）
- DaysカウンターはGlassCardとして写真の下に配置
- 既存のStreakバッジ・カード群はその下に続く

# 受け入れ条件

- [ ] ペット写真登録済みの場合、ホーム最上部に大きく表示される
- [ ] 写真未登録の場合、美しい代替デザインが表示される
- [ ] 写真の上に名前が読みやすく表示される
- [ ] 既存のDaysカウンター・記録カード群が壊れていない

# やらないこと

- 写真のフィルター加工（Soraは写真の加工をしない）
- スライドショー・アニメーション（静かさを壊す）
- 写真の切り替え機能（最新or登録写真のみ）

# 検証方法

- ホーム画面開封後1秒以内の離脱率の変化
- 「ホームをよく見る」セッション時間の変化
- Before/After でのユーザーインタビュー（「どんな印象ですか？」）
