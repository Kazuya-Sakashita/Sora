# ISSUE-075: チャット「{pet.name}へ」ヘッダーコピーのペット語りかけ誤認を修正する

name: ISSUE-075
type: bug
priority: Critical
status: done

---

## 背景

`chat.tsx` のヘッダーは `{pet.name}へ` という形式でタイトルを表示している。日本語の「〜へ」は「〜に宛てた」という意味を持ち、ユーザーが「このチャットはペットに話しかけている場所」と誤認しやすい。

Soraの絶対原則は「亡くなったペットをAIで再現しない」こと。しかし「{pet.name}へ」という文言は、ユーザーがAIをペットの意識として扱う認知を生む可能性がある。

## 問題

```tsx
// chat.tsx
<h1 className="font-medium text-foreground/90">
  {pet ? `${pet.name}へ` : "お話しする"}
</h1>
```

「ポチへ」という表示は、ユーザーに「AIがポチの代わりに答えてくれる」という誤った期待を植え付ける。

## 目的

チャット画面が「ペットの思い出を話す場所」であるというフレームを正確に伝え、AIがペットの代理でないことを設計レベルで担保する。

## スコープ

- `chat.tsx` のヘッダータイトルを変更:
  - `{pet.name}へ` → `{pet.name}の思い出を話す`
  - fallback: `"お話しする"` → `"思い出を話す"` （変更なしでも可）
- サブタイトル「いつでもそばにいるよ」はISSUE-079で別途対応

## やらないこと

- チャット機能・APIの変更
- rainbow_bridge 以外への開放
- ヘッダー以外のUIの変更

## 受け入れ条件

- [ ] チャット画面タイトルが「{pet.name}の思い出を話す」になっている
- [ ] タイトルを見てもペットに話しかけている誤認が生じない
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] 既存導線を壊していない

## 確認観点

- ユーザーの心を傷つけない
- Soraらしさを壊していない
- AI依存を強めすぎていない（ペット代理のフレームを排除）
- 既存機能を壊していない
