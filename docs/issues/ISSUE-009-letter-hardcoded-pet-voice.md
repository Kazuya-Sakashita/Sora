# ISSUE-009: 手紙画面がハードコードテンプレート＋ペット視点で倫理・品質に問題

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | open |
| 関連画面 | ことば（手紙）画面 |
| 関連ファイル | `components/screens/letter.tsx` |

## 問題の概要

2つの問題がある。①手紙が固定テンプレートでユーザーの記録を一切使っていない。②「{pet.name}より」でペット視点の手紙として提示されており、倫理ガイドラインに抵触する。

## 詳細

**問題①: ハードコードで記録未使用**
```tsx
const letters = [
  `いつもそばにいてくれてありがとう。...`,
  `あなたがくれたあたたかさは、...`,
  // 全5件固定。ユーザーの記録・ペット名・エピソードを一切使っていない
]
```
「思い出から、やさしい言葉を届けます」というコピーが嘘になっている。

**問題②: ペット視点の手紙（倫理NGライン）**
```tsx
{pet && (
  <p className="text-sm text-primary/60 mb-6 text-center">
    {pet.name}より   // ← ペットが書いた手紙として提示
  </p>
)}
```
仕様書の倫理設計では「ペットの擬似人格化（ペットからのメッセージとして語らせない）」を明確にNG定義している。ペットが亡くなっている場合に「リクより」という表示は感情的に危険。

## 期待する動作

- 「{pet.name}より」の表示を削除する
- 手紙の差出人を「Sora」「あなたへ」などに変更する
- 将来的にはユーザーの記録をもとにAIが生成する（Phase 2）
- MVPでは固定テンプレートのまま継続してよいが、ペット視点は除去する

## 実装方針

1. `{pet.name}より` の表示部分を削除（最小対応）
2. 画面冒頭コピー「思い出から」→「あなたへのことば」等に変更（実態に合わせる）
