# ISSUE-009: 手紙画面をPhase 3まで非表示化し、倫理NGの「ペット視点」を除去する

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | open |
| 関連画面 | ことば（手紙）画面 |
| 関連ファイル | `components/screens/letter.tsx`, `app/page.tsx` |

## 問題の概要

手紙画面に2つの問題がある。①固定テンプレートでユーザーの記録を使っていない。②「{pet.name}より」でペット視点の手紙として提示されており、倫理ガイドラインに抵触する。

## 詳細

**問題①: ハードコードで記録未使用**
```tsx
const letters = [
  `いつもそばにいてくれてありがとう。...`,
  `あなたがくれたあたたかさは、...`,
  // 全5件固定。ユーザーの記録・ペット名・エピソードを一切使っていない
]
```
「思い出から、やさしい言葉を届けます」というコピーが事実と異なる。

**問題②: ペット視点の手紙（倫理NGライン）**
```tsx
{pet && (
  <p className="text-sm text-primary/60 mb-6 text-center">
    {pet.name}より   // ← ペットが書いた手紙として提示
  </p>
)}
```
仕様書の倫理設計では「ペットの擬似人格化（ペットからのメッセージとして語らせない）」を明確にNG定義。ペットが亡くなっている場合に「リクより」という表示は感情的に危険。

## 仕様変更によるスコープ更新（2026-04-28）

MVP再定義（`docs/specs/sora-product-rethinking.md`）において、手紙生成機能は **Phase 3（Month 7〜）** に移動した。

**MVPでの対応方針（スコープ縮小）:**
1. `{pet.name}より` の表示を削除する（最小限の倫理修正）
2. 手紙画面をナビゲーション・ボトムタブから非表示にする
3. コンポーネントファイル自体は残す（Phase 3で全面再設計して使用）

**Phase 3での再設計方針（参考）:**
- 手紙の主体は「あなたがペットに伝えたかった言葉」。「ペットからの手紙」ではない
- AIがユーザーの記録データを読んで言語化する
- 生成前に「今の気持ちで大丈夫ですか？」の確認UI

## 実装方針

1. `{pet.name}より` の表示部分を削除
2. `app/page.tsx` のルーティングから手紙画面を除外
