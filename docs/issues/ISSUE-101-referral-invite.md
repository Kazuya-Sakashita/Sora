# ISSUE-101: 友人招待・リファラルリンク機能を実装する

name: ISSUE-101
type: feature
priority: High
status: open

---

## 概要

設定画面に「友人を招待する」機能を追加し、招待URL付きのシェアメッセージを生成する。招待経由の登録数をトラッキングし、将来の特典機能の基盤を作る。

## 背景

Growth Loop 2（グリーフ口コミループ）と Loop 3（コミュニティループ）を仕組み化するには、既存ユーザーが友人に紹介しやすい機能が必要。「Soraがあってよかった」という感情が高まるタイミング（ペットロス直後・記念日）に自然に紹介できる導線を用意する。

K-factorを1.0以上にするために、紹介コストを極限まで下げる必要がある。

## 目的

既存ユーザーが友人・家族にSoraを紹介するときの摩擦をゼロにし、口コミによる自然流入を仕組み化する。

## スコープ

- 設定画面に「友人に紹介する」セクションを追加
- 招待URL: `https://sora-app.jp/invite/{referralCode}`
- referralCodeはユーザーごとにユニークな8文字コードをDBに保存
- シェアメッセージのプリセット:
  - 「ペットとの毎日を残せるアプリSoraを使っています。生前から記録しておくと、大切な思い出になるよ。{URL}」
- Web Share API + クリップボードコピーに対応
- 招待経由で登録されたユーザーのreferrerIdをDBに保存（将来の特典付与に備えて）
- 招待した人数を設定画面に表示（「●人に紹介済み」）

## 非スコープ

- 招待特典（Sora+無料期間など）は今回実装しない
- 招待の有効期限設定
- メール招待

## API変更

あり
- `GET /api/users/referral` — 自分のreferralCodeを取得または生成
- `POST /api/users/referral/register` — 招待経由登録時にreferrerIdを記録
- openapi.yaml を先に更新すること
