# ISSUE-045: 週次リマインダープッシュ通知

name: ISSUE-045
type: feature
priority: Medium
status: open

---

## 概要

毎週月曜朝に「先週の記録件数とストリーク」を通知し、D7・D30 リテンションを改善する。ストリークが途切れた翌日は専用の復帰通知を送る。

## 背景

現状のプッシュ通知は毎日同じ「今日を残しませんか」のみ。通知内容がパーソナライズされていないため、通知を無視・オフにするユーザーが多い。週次サマリーは「振り返り」の動機を与え、リエンゲージメントに効く。

## 目的

- D7・D30 リテンション改善
- 習慣が途切れたユーザーの復帰率向上

## スコープ

- `app/api/push/weekly/route.ts` を新規作成
  - CRON_SECRET 認証
  - 全 PushSubscription ユーザーに対して実行
  - 先週（月〜日）の記録件数を集計：`prisma.memory.count({ where: { date: { gte, lte } } })`
  - 現在のストリーク計算：既存 `calcStreak()` を流用
  - 通知パターン：
    - 先週1件以上：「先週は${n}件記録しました🎉 今週も続けましょう」
    - 先週0件かつストリーク > 0：「ストリークが途切れる前に、今日記録しませんか🔥」
    - 先週0件かつストリーク = 0：「${pet.name}の今週を残しませんか」
  - `web-push` で送信（既存 `sendPushNotification` を流用）
- `vercel.json` または `app/api/push/weekly/route.ts` の設定ファイルに CRON 設定追加
  - スケジュール：`0 23 * * 0`（日曜22時 UTC = 月曜8時 JST）

## 非スコープ

- ユーザーごとの通知頻度設定
- 通知タイミングのカスタマイズ UI

## API変更

- なし
