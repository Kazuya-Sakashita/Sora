## 問題の概要

現在の写真投稿がローカルの一時URLや未実装のままになっている。Supabase Storageを使ったクラウドへの写真アップロードが必要。

## 詳細

記録機能の核心は写真投稿。写真がクラウドに永続化されなければ、ISSUE-001（データ永続化）を完了しても写真だけ消える状態になる。

## 期待する動作

- ユーザーがデバイスのカメラロールから写真を選択できる
- 選択した写真がSupabase Storageにアップロードされる
- アップロードされた写真の公開URLが記録に紐付いて保存される
- タイムラインで写真が表示される
- アップロード中はプログレス表示またはスケルトンUIが表示される
- 写真の容量制限: 1枚10MB以内（無料プラン）

## 実装方針

1. Supabase Storageのバケット設定
   - バケット名: `memories`
   - アクセス制御: 認証済みユーザーのみ自分の写真にアクセス可能
   - RLSポリシー: `auth.uid() = user_id`
2. 記録入力コンポーネントにファイルインプットを実装
   ```tsx
   // input[type="file"] accept="image/*"
   // onChange で supabase.storage.from('memories').upload() を呼ぶ
   ```
3. アップロード後のURL取得: `getPublicUrl()` または署名付きURL
4. 記録データにphoto_urlを含めてDBに保存
