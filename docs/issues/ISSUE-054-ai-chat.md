# ISSUE-054: 「はなす」画面をClaude APIで実際の対話に置き換える

name: ISSUE-054
type: feature
priority: High
status: done

---

## 概要

ロスケアモード専用の「はなす」画面を、固定ランダム返答からClaude APIによるリアルタイム対話に置き換える。

## 背景

現状の「はなす」はハードコードされた8パターンをランダムに返すだけで、ユーザーが何を送っても同じ文言が返る。ロスケア中の飼い主にとって「話を聞いてもらえている」感覚がなく、離脱の原因になっている。ISSUE-009でPhase 3まで非表示としていたが、ISSUE-033でrainbow_bridge時のみ表示を解放済み。AIインフラ（lib/ai.ts）も整ったためリアル実装に移行する。

## 目的

- ロスケアユーザーが「このアプリは私の話を聞いてくれている」と感じられる対話を実現する
- ペットとの思い出・気持ちを自然に言語化できる場を提供する

## スコープ

- `POST /api/pets/[petId]/chat` を新規追加
  - リクエスト: `{ messages: { role: "user" | "assistant", content: string }[], petName: string }`
  - レスポンス: `{ reply: string }`
  - システムプロンプト: ペット名・ロスケアモードの文脈を設定、穏やか・寄り添う語り口
- 会話履歴はフロントのstate管理（ページリロードでリセット、永続化はPhase 2）
- 直近の会話を最大10往復コンテキストとして渡す
- ストリーミング不要（通常のAPI呼び出し）
- レート制限: 1ユーザー・1分間に10メッセージまで
- エラー時はインラインメッセージで静かに通知（「少し待ってから、もう一度試してみてください」）

## 非スコープ

- 会話履歴のDB永続化（Phase 2）
- ストリーミング応答
- ペット視点の返答（倫理NG、ISSUE-009の判断を維持）

## API変更

- あり
- `POST /api/pets/[petId]/chat` を openapi.yaml に追加

## 受け入れ条件

- [ ] ユーザーのメッセージに対してClaude APIが文脈を踏まえた返答を返す
- [ ] ペット名が会話の文脈に含まれる
- [ ] 絵文字・過度な共感・ペット視点の発言が含まれない
- [ ] レート制限超過時に適切なエラーメッセージが表示される
- [ ] APIエラー時に会話が壊れず静かに通知される
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: メッセージ送信→Claude応答が表示される
- 正常系: 複数往復で会話の文脈が維持される
- 異常系: APIエラー時にインラインエラー表示
- 文体確認: 穏やか・寄り添う語り口・ペット視点なし
- UI確認: 既存のチャットUIと自然に統合されている
