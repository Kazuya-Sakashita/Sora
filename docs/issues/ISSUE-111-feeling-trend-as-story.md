# 概要

現在の感情トレンド画面（グラフ表示）を、「今月はこんな心でした」という言語化されたサマリーとして見せる形に変える。数値・グラフより言葉の方が、感情の受け皿として機能する。

# 背景

感情トレンドのグラフは「データ分析」の文脈で動く。ペットロスの感情を棒グラフで見るのは、感情と向き合うというより感情を観察するUXになっている。「今月いちばん多かった気持ちは悲しみでした」という言葉の方が、「あ、そうだったんだ」と受け取れる。

# 解決したい課題

- グラフが「管理ツール」の印象を強める
- 数値で感情を見ることへの抵抗感
- 週次の気持ちの変化が「自分ではわかっていない」ことがある

# ユーザー価値

「今月の自分の心を、誰かが言葉にしてくれた」という感覚。自己理解の補助。

# 実装方針

**月次サマリー表示（感情トレンド画面）:**
- 「今月の{name}との時間」セクション
- テキスト：「今月、{top_tag_label}な日が{n}日ありました」
- AIによる1〜2文の柔らかいサマリー（例：「今月は穏やかな日が多かったようです。{name}のことを思いながら、ゆっくり過ごせた月だったかもしれません」）
- 件数の多い気持ちタグ3つを並べる（グラフではなくバッジ形式）

**Rainbow Bridge期の特別表示:**
- 「あれから{n}日が経ちました」という節目言語
- 「今月の気持ちは{tag}が多かったようです。それでよかったと思います」
- 「変わってきている感覚はありますか？」という静かな問いかけ（1文のみ）

**グラフは「詳細を見る」として折り畳む:**
- 詳細派ユーザーのためにグラフはアコーディオン内に
- デフォルト表示はテキストサマリーのみ

# 受け入れ条件

- [ ] 感情トレンド画面の初期表示がテキストサマリーになっている
- [ ] AIが今月の気持ちを1〜2文で要約する
- [ ] グラフは「詳細を見る」ボタンで展開できる
- [ ] Rainbow Bridge期は節目の言及とやさしい問いかけが含まれる
- [ ] サマリーに「前を向きましょう」「改善しましょう」等の文言がない

# やらないこと

- 感情スコアリング（「あなたのグリーフスコアは〇〇です」）
- 気持ちの分布を「理想値」と比較する
- 「心配な傾向があります」という危険シグナル系メッセージ（AIには診断させない）

# 検証方法

- 感情トレンドタブの閲覧率・滞在時間の変化
- テキストサマリー vs グラフ、どちらで止まるか（スクロール深度）
- 定性：「感情を振り返る体験が変わったか？」
