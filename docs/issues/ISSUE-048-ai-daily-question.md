# ISSUE-048: AIデイリー問いかけをホーム画面に実装する

name: ISSUE-048
type: feature
priority: Critical
status: open

---

## 概要

毎朝ホーム画面に「今日の問いかけ」カードをAI生成で表示し、記録開始のトリガーを作る。

## 背景

現状、アプリを毎日開く内発的な理由がない。プッシュ通知は習慣化に効果が薄く、「今日何を記録するか」をユーザー自身が考える負荷が離脱につながっている。DAU/MAU比が低い根本原因。

## 目的

- 毎日アプリを開く理由を設計する
- 記録開始のハードルをゼロにする
- North Star Metric（週3回記録率）を改善する

## スコープ

- ホーム画面上部に「今日の問いかけ」カードを追加
- Claude APIに以下を渡して1行の問いかけを生成
  - 直近7日間の記録タイトル（最大5件）
  - ペットの名前・種類・年齢
  - 現在の月（季節コンテキスト）
- 日付変更（00:00）で再生成。同日中は同じ内容を返す
- カードタップで記録作成画面へ遷移（問いかけ文をタイトルに引き継ぎ）
- 生成失敗時はデフォルト文を表示（「今日の{ペット名}はどんな様子でしたか？」）

## 非スコープ

- 問いかけのパーソナライズ深化（Phase 2）
- 複数の問いかけ候補からの選択

## API変更

- `GET /api/pets/[petId]/daily-question` を新規追加
  - レスポンス: `{ question: string, generatedAt: string }`
  - キャッシュ: 当日中は同じ値を返す（DB or KVに保存）

## 受け入れ条件

- [ ] ホーム画面に「今日の問いかけ」カードが表示される
- [ ] 毎日異なる問いかけが生成される
- [ ] 記録がない日も問いかけが表示される
- [ ] カードタップで記録画面に遷移し問いかけ文が引き継がれる
- [ ] API失敗時にデフォルト文が表示される
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: ホームに問いかけカード表示
- 正常系: タップで記録画面遷移・テキスト引き継ぎ
- 異常系: API失敗時のデフォルト表示
- 境界値: 記録が0件のユーザーでも表示される
