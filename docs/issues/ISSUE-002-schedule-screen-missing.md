# ISSUE-002: 予定管理画面が未実装（MVPの必須機能）

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | open |
| 関連画面 | なし（新規作成） |
| 関連ファイル | `components/screens/`（新規）, `lib/app-context.tsx`, `app/page.tsx` |

## 問題の概要

仕様書のMVPに必須の「予定管理（通院・トリミング・ワクチン・記念日）」画面が未実装。

## 詳細

V0生成のUIには予定管理画面が含まれていない。ホームの4つのクイックアクションにも予定管理がない。この機能はユーザーが「日常的に開く理由」を作る重要な機能で、MVPに必須。

## 期待する動作

- 通院・トリミング・ワクチン・記念日の4種類の予定を追加できる
- 追加した予定が一覧表示される
- ホーム画面から予定管理画面に遷移できる

## 実装方針

- `components/screens/schedule.tsx` を新規作成
- 予定のデータ型を `app-context.tsx` に追加
- Screen型に `schedule` を追加
- ホームのクイックアクションに「予定を管理する」を追加（現在4つ→最大5つ or 並べ替え）
- UIは既存のGlassCard・デザインシステムに合わせる

- [ ] 予定の追加・一覧表示が動く
- [ ] ホームから遷移できる
- [ ] データが永続化される（ISSUE-001完了後に対応）
- [ ] 既存画面への影響なし確認済み
