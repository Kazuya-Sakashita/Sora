# ISSUE-061: チャット履歴をlocalStorageで永続化する

name: ISSUE-061
type: improvement
priority: Critical
status: done

---

## 背景

「記録を通して、もう一度つながれる体験」がSoraの核。しかしチャット画面はReact stateのみで会話を管理しており、画面を離れると履歴が完全に消える。深い悲しみを打ち明けた翌日にAIが何も覚えていない体験は、プロダクトの核心約束への裏切りになる。

## 問題

`chat.tsx` の messages は `useState` のみ。ナビゲーションのたびに「ここにいるよ。無理しなくて大丈夫」の初期メッセージにリセットされる。「覚えていてくれている感」がゼロ。

## 目的

ペットごとにチャット履歴をlocalStorageに保存し、次回起動時に復元する。「昨日話したこと、ちゃんとここにある」という継続性を担保する。

## スコープ

- `sora:chat-{petId}` キーで localStorage に最新20件を保存
- マウント時に復元（timestamp は Date 変換）
- メッセージ更新のたびに自動保存
- ヘッダーに「会話をリセット」ボタンを追加（使ったときだけ表示）
- 保存失敗（storage full 等）はサイレントに無視

## やらないこと

- チャット履歴のDB永続化
- チャット履歴のPLUS限定化
- 履歴のエクスポート機能

## 受け入れ条件

- [ ] チャット後にホームへ戻り再度開くと履歴が残っている
- [ ] リセットボタンで会話が初期状態に戻る
- [ ] 別ペットに切り替えると別の履歴が読み込まれる
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] gstack で動作確認済み
