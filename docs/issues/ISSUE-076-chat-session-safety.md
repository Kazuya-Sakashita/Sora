# ISSUE-076: チャット長時間セッションに休憩提案メッセージを挿入する

name: ISSUE-076
type: improvement
priority: Critical
status: done

---

## 背景

`chat.tsx` には現在、セッション長さの制限がない。rainbow_bridge期のユーザーは急性悲嘆状態にある場合が多く、1時間以上AIと話し続けることが技術的に可能な状態だ。

グリーフカウンセリングの観点では、感情的なAI対話を無制限に行うことは、現実回避・AI依存の強化につながるリスクがある。`chat.tsx` の専門サポートリンク（HeartHandshakeボタン）は存在するが受動的で、長時間利用後に自動で表示されることはない。

## 問題

```tsx
// chat.tsx — メッセージ数に関わらず無制限に会話が続く
// 専門サポートリンクはヘッダーボタンからのみアクセス可能
```

ユーザーが10往復以上会話を続けても、Soraからの「今日はここで区切りましょうか」という提案は一切ない。

## 目的

長時間の使用を禁止するのではなく、自然な「区切り」を提案することで、ユーザーが適切な休息を取れるよう設計する。専門家のサポートへの導線も自然に示す。

## スコープ

- ユーザーのメッセージ数（user role のメッセージ）が8件に達したとき、次のAI返答の後に自動でシステムメッセージを1件挿入する
- システムメッセージの形式:
  - role: `assistant`
  - content: `今日はたくさん話してくれましたね。少し休んで、また話しましょう。`（話しかけ口調はSoraのルールに沿う）
  - 通常のAIバブルと区別しない（同じUIで表示）
- このメッセージは1セッションにつき1回のみ挿入（2回目以降は挿入しない）
- メッセージ数カウントは `messages.filter(m => m.role === "user").length` で計算
- localStorage に保存するチャット履歴にはこのメッセージも含む（削除不要）

## やらないこと

- 強制終了・入力ロック
- メッセージ送信のブロック
- 8件以降の会話を禁止すること
- セッション時間タイマーの実装

## 受け入れ条件

- [ ] ユーザーが8件目のメッセージを送った後、AIの返答の後に休憩提案メッセージが表示される
- [ ] 同一セッション内で休憩提案は1回しか表示されない
- [ ] 休憩提案後もユーザーは自由に会話を続けられる
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] 既存チャット機能を壊していない

## 確認観点

- ユーザーの心を傷つけない（強制ではなく提案のトーン）
- Soraらしさを壊していない
- AI依存を強めすぎていない（健全な使用習慣を促す）
- 既存機能を壊していない
