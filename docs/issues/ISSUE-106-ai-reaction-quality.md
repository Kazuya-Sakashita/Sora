# 概要

記録保存後のAI反応（`POST /api/pets/{petId}/memories/{memoryId}/reaction`）を、「記録の中身を本当に読んだ」と感じさせる品質に引き上げる。現在の汎用的な2〜3文の反応では「誰に送っても同じ言葉」になっており、初回体験で最も重要な「受け取ってもらえた」感覚が生まれていない。

# 背景

テストユーザーが「心が動かない」「記録する意味が弱い」と感じる原因の中核はここにある。記録を保存した直後のAI反応は、アプリ全体の「約束」を体現する瞬間であり、ここが汎用的では初回Retentionが生まれない。

# 解決したい課題

- 「ポチが公園を走っていました」という記録に「素敵ですね」だけ返してくる
- 写真があってもなくても同じ反応になる
- Rainbow Bridge期のトーンが生前と同じになる
- 同じユーザーが同じような記録を続けると反応パターンが繰り返される

# ユーザー価値

「Soraに読んでもらえた」という確かな感覚。記録を続けるいちばんの理由になる。PMFスコア（「Soraがなくなったらとても残念」）の主要因。

# 実装方針

**プロンプト改善の方向性:**

1. **記録の具体的な言及を必ず含める**
   - タイトルの固有名詞・動詞を使った応答
   - 例：「公園を走っていたんですね」「今日はゆっくりした一日だったのですね」

2. **写真有無で応答を分ける**
   - 写真あり：「その瞬間を写真に残せたんですね」などの言及
   - 写真なし：テキストだけの記録の良さを肯定する

3. **気持ちタグを反応に織り込む**
   - happy: 「うれしそう」「楽しかったのが伝わります」
   - worried: 「心配していたんですね。大切に思っているから」
   - rainbow_bridge・sad: 「悲しい気持ちを、ここに残してくれてありがとう」

4. **Rainbow Bridge期は静かに変える**
   - 過去形・懐かしむトーン
   - 「今日もここに来てくれましたね」という受容
   - 問いかけより「一緒にいる」表現

5. **連続記録への言及（10件・50件節目）**
   - 「{n}回目の記録ですね」と蓄積を肯定

**技術的実装:**
- `POST /api/pets/{petId}/memories/{memoryId}/reaction` のプロンプトを再設計
- pet.name / memory.title / memory.description / memory.moodTag / memory.photoUrls.length / pet.status をすべてプロンプトに含める
- 反応の長さ：1〜3文（短い方がいい。長いと「読んでない」感が出る）
- 反応のバリエーション：同じ条件で複数パターンを生成し、ランダム選択

# 受け入れ条件

- [ ] 記録タイトルに含まれる内容が反応に反映されている（固有名詞・動詞）
- [ ] 写真あり記録と写真なし記録で反応が異なる
- [ ] Rainbow Bridge期の反応は生前より静かなトーンになっている
- [ ] 同じユーザーの連続記録で同一の反応パターンが繰り返されない
- [ ] プロンプトに「前を向いて」「頑張って」「大丈夫です」などを含まない
- [ ] 1〜3文以内に収まっている

# やらないこと

- ペットの口調・性格を模倣した「ペット視点の反応」（世界観を壊す）
- 医療・健康アドバイス
- 「記録を増やしましょう」などの行動指示

# 検証方法

- 記録直後のアプリ継続率（24h以内に再起動したか）
- 記録後の滞在時間（反応画面で止まっているか）
- A/Bテスト：旧プロンプト vs 新プロンプトで翌日Retention比較
