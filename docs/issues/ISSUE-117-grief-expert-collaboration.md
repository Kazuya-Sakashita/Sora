# 概要

ペットロスのグリーフケア専門家（カウンセラー・獣医師・心理士）とコラボし、Soraのコンテンツ・AI応答の品質を高める。専門家の監修を受けることで信頼性と安全性を確保する。

# 背景

ペットロスは深刻な心理的影響を伴うが、Soraは「AIが診断しない」方針を取っている。しかし、AIの言葉が不適切な場合のリスク（「乗り越えましょう」等）は現状のプロンプト設計だけでは完全には排除できない。専門家の知見を組み込むことで品質の底上げと信頼性担保ができる。

# 解決したい課題

- AIの言葉が専門的観点から適切かどうかを検証する仕組みがない
- 「ペットロスに寄り添うアプリ」としての信頼性の根拠が薄い
- 深刻なグリーフ状態のユーザーへの適切なリソース案内がない

# ユーザー価値

「専門家が監修したアプリ」という安心感。深刻な状態のときに適切なサポートへ接続される。

# 実装方針

**Phase 1: コンテンツ監修（対応優先度: 低）**
- グリーフケア専門家2〜3名にAIプロンプト・コピーのレビューを依頼
- NG表現リスト・OK表現リストを整備
- 「監修：○○先生」のクレジットを適切な箇所に表示

**Phase 2: 専門家リソース案内（任意）**
- 深刻なグリーフシグナル（記録頻度の急減・特定の感情タグの連続）を検知した場合
- 「気持ちが辛いときは、専門家への相談も一つの選択肢です」という静かな案内
- 外部リソースへのリンク（ペットロスホットライン等）

**Phase 3: 専門家向けポータル（将来）**
- グリーフカウンセラーがSoraを患者に勧めるためのリソース
- B2B展開の一環

**実装上の注意:**
- Soraは診断ツールではないことを明示し続ける
- 専門家リソース案内は強制・プッシュしない（ユーザーが求めたときに応じる形）
- 危機介入機能（自傷・自殺リスク）は実装しない（スコープ外・責任問題）

# 受け入れ条件

- [ ] AIプロンプト・コピーの専門家レビューが完了している
- [ ] NG表現リストが `docs/ai-guidelines.md` に整備されている
- [ ] 深刻なグリーフシグナル時に外部リソースへの案内が表示される（任意フロー）

# やらないこと

- Soraをグリーフ診断ツールとして位置づける
- 危機介入・自傷リスク検知の実装
- 専門家との有償セッション機能（スコープ外）

# 検証方法

- 専門家監修後のAI応答品質の定性評価
- 外部リソース案内のクリック率・その後のリテンション
- ユーザーの「このアプリを信頼している」スコアの変化
