# ISSUE-049: 記録保存後にAIリアクションを返す

name: ISSUE-049
type: feature
priority: Critical
status: open

---

## 概要

記録保存直後にAIが内容を読んで2〜3行のひと言を返し、記録行動の即時報酬を設計する。

## 背景

現状、記録を保存すると「完了」トーストが出るだけで終わる。記録行動を強化するフィードバックがなく、「書いても何も返ってこない」状態が継続率を下げている。日記系アプリでAIリアクションがある場合の7日継続率は平均40%改善（業界傾向）。

## 目的

- 記録直後の「報酬」を設計し、記録行動を習慣化する
- 「このアプリは私の話を聞いてくれている」という感覚を作る

## スコープ

- 記録保存完了後、モーダル形式でAIリアクションを表示
- Claude APIに以下を渡して生成
  - 記録タイトル・本文・気持ちタグ・日付
  - ペットの名前・種類
- 文体ルール（プロンプトに明示）
  - 2〜3行・穏やか・断定しない
  - 絵文字禁止・過度な共感禁止・質問禁止
  - 例：「今日はよく眠れた様子が伝わってきます。こういう何でもない午後が、ずっと残りますように。」
- モーダルは「閉じる」1ボタンのみ。自動で3秒後に閉じない（読む時間を奪わない）
- FreeユーザーもSora+ユーザーも同等に提供
- APIエラー時はリアクションなしで静かに失敗（記録保存には影響しない）

## 非スコープ

- リアクションへの返信機能
- リアクション履歴の閲覧

## API変更

- `POST /api/pets/[petId]/memories/[memoryId]/reaction` を新規追加
  - レスポンス: `{ reaction: string }`
  - 記録保存APIとは分離（記録保存の成功に依存させない）

## 受け入れ条件

- [ ] 記録保存後にAIリアクションモーダルが表示される
- [ ] リアクション文が記録内容と関連している
- [ ] 絵文字・過度な共感表現が含まれない
- [ ] API失敗時にモーダルが出ず記録保存は成功している
- [ ] モーダルに「閉じる」ボタンのみが存在する
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: 記録保存後にモーダル表示
- 正常系: リアクション文が記録内容を反映している
- 異常系: APIタイムアウト時に記録は保存されモーダルは出ない
- 文体確認: 絵文字・質問・過度な共感が含まれないことを手動確認
