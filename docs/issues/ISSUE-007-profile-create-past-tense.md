# ISSUE-007: プロフィール登録画面のコピーが過去形でロス後前提になっている

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | critical |
| ステータス | done |
| 関連画面 | プロフィール登録画面 |
| 関連ファイル | `components/screens/profile-create.tsx` |

## 問題の概要

「どんな子でしたか？」「好きだったこと」「思い出」など、ペットがすでに亡くなっている前提の過去形コピーが全面に使われている。

## 詳細

```
現状のコピー（問題箇所）:
- タイトル: 「どんな子でしたか？」← 過去形
- サブコピー: 「思い出せる範囲で大丈夫ですよ」← 亡くなった後に思い出すニュアンス
- ラベル: 「好きだったこと」← 過去形
- ラベル: 「思い出を少しだけ教えてください」← 過去形
- プレースホルダー: 「いちばん印象に残っていること...」← 回想ニュアンス
```

生きているペットを初めて登録するユーザーが「どんな子でしたか？」と聞かれると「ん？うちの子まだ生きてるけど？」となる。生前利用の最初のステップでこのコピーが出ると、アプリの世界観が崩壊する。

## 期待する動作

生前ユーザーが自然に入力できる文言に変更する。

```
修正案:
- タイトル: 「あなたの大切な子を教えてください」
- サブコピー: 「少しずつで大丈夫です」（現状のままでOK）
- ラベル「好きだったこと」→「好きなこと」（現在形）
- ラベル「思い出を少しだけ教えてください」→「どんな子か教えてください」
- プレースホルダー: 「得意なこと、よくする行動...」
```

## 実装方針

`profile-create.tsx` 内のテキスト文字列を差し替えるだけ。ロジックの変更なし。
