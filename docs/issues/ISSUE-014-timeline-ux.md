# ISSUE-014: タイムライン画面を写真主体のUXに改善する

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | open |
| 関連画面 | タイムライン画面 |
| 関連ファイル | `components/screens/timeline.tsx` |

## 問題の概要

現在のタイムライン画面が「記録の価値」を感情的に伝えるUXになっていない。写真を主体とした、スクロールしたくなる美しいタイムラインが必要。

## 詳細

タイムラインはSoraの中核体験の一つ。「こんなに一緒にいたんだ」という発見と、「もっと記録したい」という動機が生まれる設計が求められる。

現状の問題:
- 写真のサイズが小さい・目立たない
- 記録の「感情」が伝わるデザインになっていない
- 「去年の今日」等の感情トリガーがない
- スクロールが単調

## 期待する動作

**カードデザイン:**
- 写真を全幅または大きめに表示（16:9 または 4:3 比率維持）
- 気持ちタグを写真の下にアイコン + ラベルで表示
- メモがある場合は折りたたみ可能なテキストで表示
- 日時は控えめに（年月日のみ、時刻は省略）

**スクロール体験:**
- 無限スクロール（または「もっと見る」ボタン）
- 月ごとのセクションヘッダー（「2026年3月」等）

**フィルタリング:**
- 気持ちタグ別フィルター
- 月別フィルター

## 実装方針

1. `timeline.tsx` のカードコンポーネントを写真主体に再設計
2. 月別セクションヘッダーを追加
3. 気持ちタグの表示を追加
4. ISSUE-005（空状態）と合わせて実装
