# ISSUE-074: alive期「今月のひとこと」の表示条件を緩和する

name: ISSUE-074
type: bug
priority: High
status: open

---

## 背景

ISSUE-060で実装した alive期「今月のひとこと」カードは、`day >= 26`（月末5日間のみ）という条件で表示される。この条件は「月のふりかえり」という意図を表しているが、実際には月の95%のユーザーがこのカードを見られない。

月次AIメッセージはSoraの差別化機能の一つで、「AIが自分のペットのことを考えてくれている」体験を提供する。月末限定では継続利用のフック強度が著しく低い。

## 問題

```tsx
// home.tsx (ISSUE-060実装部分)
const day = new Date().getDate()
// ...
{day >= 26 && aliveMonthlyMessage && (
  <AliveMonthlyMessageCard message={aliveMonthlyMessage} />
)}
```

月の大半（1〜25日）はカードが表示されない。ユーザーが「今月のひとこと」を目にする機会が極めて限られる。

## 目的

表示条件を緩和し、記録が一定数ある月は月中でもカードを表示できるようにする。ユーザーが「AIがいつも見てくれている」と感じられる頻度を上げる。

## スコープ

- 表示条件を `day >= 26` から以下のいずれかに変更:
  - **案A（推奨）**: `currentMonthRecordCount >= 3` — 今月3件以上記録があればいつでも表示
  - 案B: `day >= 10` — 月の前半以降は常時表示
  - 採用案: **案A**（記録行動と報酬を直結させることでエンゲージメント効果が高い）
- `currentMonthRecordCount` は既存の `memories` から当月分をフィルタして算出
- メッセージ再生成の頻度: 月1回のキャッシュは維持（条件変更後も同じキーで管理）

## やらないこと

- AIメッセージ生成ロジックの変更
- メッセージの複数回生成
- 表示UIの変更

## 受け入れ条件

- [ ] 今月3件以上記録があればaliveMonthlyMessageカードが表示される
- [ ] 今月2件以下の場合はカードが表示されない（記録促進のため）
- [ ] 月次メッセージキャッシュ（既存ロジック）が正常動作する
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] 既存導線を壊していない

## 確認観点

- 記録3件という閾値がユーザーに圧力にならないか（記録へのハードルを上げすぎない）
- メッセージ品質が維持されているか（月初に3件達成した場合でも適切な内容か）
- 既存機能を壊していない
