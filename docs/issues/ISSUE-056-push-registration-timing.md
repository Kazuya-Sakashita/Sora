# ISSUE-056: Push通知登録タイミングをActivation時に前倒しする

name: ISSUE-056
type: improvement
priority: High
status: done

---

## 概要

Push通知の登録促進が「設定画面」でしか行われていない。あの日通知・週次リマインダー・月次レター到着通知はすべて Push 依存なのに、ユーザーが設定画面に到達しなければ永遠に登録されない。

## 背景

現状のフロー:
```
オンボーディング → 初回記録 → ホーム（Push登録の促進なし）→ 設定 → 通知トグル
```

Push通知を軸にした継続施策（ISSUE-045週次・ISSUE-053あの日・ISSUE-051月次レター）が全て実装済みにもかかわらず、登録率が低いと施策の効果がゼロになる。ユーザーの多くは設定画面に自発的に行かない。

## 目的

- Push通知登録率を上げる
- あの日通知・月次レターが意図した効果を発揮できるようにする

## スコープ

- 初回記録完了後（`first-record.tsx` のcompletion画面）にPush通知の許可を促すステップを追加
  - 「記録を忘れた日に、そっとお知らせします」というコピーで許可を求める
  - 「今は後で」を選べるようにする（強制しない）
  - すでに許可済みの場合はスキップ
- ホーム画面の初回表示時（初回のみ）にソフトなバナーを表示
  - GlassCard スタイルの目立たないバナー
  - 「通知の設定はこちら →」で設定画面に誘導
  - 1回表示したら `localStorage` で非表示フラグを立てる
  - Push通知が denied/granted の場合は表示しない（default のみ）

## 非スコープ

- Push通知の許可を強制するモーダル（ユーザー体験を損なう）
- オンボーディング中の通知許可（ペットを知る前の段階では早すぎる）

## API変更

- なし

## 受け入れ条件

- [ ] 初回記録完了後のページに通知許可の促しが表示される
- [ ] 「今は後で」で進めた場合、ホームに遷移できる
- [ ] ホーム初回表示時に通知誘導バナーが1回だけ表示される
- [ ] 通知許可済み・拒否済みの場合は促しが表示されない
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: 初回記録完了→通知許可プロンプト表示
- 正常系: 「今は後で」→ホーム遷移
- 正常系: ホーム初回表示→バナー表示
- 境界値: 2回目のホーム表示→バナー非表示
- 境界値: 通知許可済みの場合→促し非表示
