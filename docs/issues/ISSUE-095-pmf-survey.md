# ISSUE-095: アプリ内PMFサーベイを実装する

name: ISSUE-095
type: feature
priority: High
status: open

---

## 概要

記録5件以上のユーザーに対して「このアプリがなくなったら？」の1問サーベイをアプリ内で1回だけ表示し、PMF達成度を測定する。

## 背景

Sean Ellisテストによると「非常に残念」と回答するユーザーが40%以上でPMF達成とみなせる。Soraが成長投資に進むべきか判断するために、この計測が最初に必要。現状は実ユーザーの満足度を測る仕組みがない。

## 目的

PMF達成度を数値で把握し、Stage 2（Activation最適化）〜Stage 4（Distribution）への投資判断を根拠づける。

## スコープ

- 記録件数が5件以上になったタイミングで、ホーム画面に1回だけモーダルを表示
- 設問: 「Soraが使えなくなったら、どう感じますか？」
- 選択肢3択:
  - とても残念（非常に残念）
  - 少し残念
  - 特に困らない
- 回答をlocalStorageに保存（再表示しない）
- 設定画面からいつでも回答確認・再回答できるリンクを追加
- API不要（MVP段階はローカル保存のみ）

## 非スコープ

- サーバーへの集計送信（将来対応）
- 複数設問のNPS調査
- 回答に応じたUI分岐

## API変更

なし

## 受け入れ条件

- [ ] 記録5件達成後、ホームに1回だけモーダルが表示される
- [ ] 3択から1つ選ぶと閉じて再表示されない
- [ ] アプリを再起動しても再表示されない
- [ ] 設定画面に「アプリへのご意見」リンクがある

## テスト観点

- 正常系: 5件記録後にモーダル表示、選択後に消える
- 境界値: 4件では表示されない、6件でも1回だけ
- 永続化: localStorage保存後は再表示しない
- UI動作: モーダルが既存画面を壊さない
