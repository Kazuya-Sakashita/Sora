# ISSUE-096: ホームのFABからクイック記録を1画面で完結させる

name: ISSUE-096
type: feature
priority: High
status: open

---

## 概要

ホームのFABボタンから写真選択→タイトル→保存をボトムシート1枚で完結させ、記録までのタップ数を削減する。

## 背景

現状の記録フローはホーム → タイムライン画面遷移 → 記録ボタン → 入力フォーム画面と2回の画面遷移が発生する。Hook Modelの「Action」ステップが重く、衝動的な記録（散歩中にかわいい瞬間を撮った直後など）でのドロップが発生していると推測される。

Activation最適化の観点から、「思い立ったらすぐ残せる」体験が"Aha Moment"の早期達成につながる。

## 目的

ホームから3タップ以内で記録を完了できるようにし、記録頻度と初回記録完走率を上げる。

## スコープ

- ホーム画面のFABボタンをタップすると、ボトムシートが開く
- ボトムシート内で以下を完結:
  - 写真選択（任意・最大3枚）
  - タイトル入力（必須）
  - 保存ボタン
- 保存後は既存の記録完了フィードバック（AIリアクション）を表示
- 詳細入力（説明・日付・カテゴリ・気持ちタグ）はタイムラインから後から編集可能な旨をUIで示す

## 非スコープ

- ボトムシートへの全フィールド詰め込み（シンプルさ優先）
- タイムライン画面の既存記録フォームの変更

## API変更

なし（既存の POST /api/pets/{petId}/memories を使用）

## 受け入れ条件

- [ ] ホームのFABタップでボトムシートが開く
- [ ] 写真なし・タイトルのみで保存できる
- [ ] 保存後にAIリアクションが表示される
- [ ] タイムライン画面の記録フローは従来通り動作する
- [ ] キーボード表示時にボトムシートが隠れない

## テスト観点

- 正常系: タイトルのみで保存、写真つきで保存
- 異常系: タイトル空で保存しようとするとバリデーションエラー
- UI動作: ボトムシートの開閉アニメーション、キーボード対応
