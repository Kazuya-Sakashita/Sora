# ISSUE-003: オンボーディングを新しく迎えたペットのオーナー向けに全面再設計する

| 項目 | 内容 |
|------|------|
| 優先度 | high |

| 関連画面 | オンボーディング画面 |
| 関連ファイル | `components/screens/onboarding.tsx` |

## 問題の概要

現在のオンボーディングがペットロス専用アプリとして見える可能性がある。MVP再定義に伴い、**メインターゲットを「新しくペットを迎えた人」に設定し**、オンボーディングを全面再設計する。

## 詳細

仕様再考（`docs/specs/sora-product-rethinking.md`）でメインターゲットが更新された。

**旧:** ペットロスを想定した「亡くなった後のために記録を残す」文脈  
**新:** 「今日子犬・子猫を迎えた。最初の日から全部記録したい」という文脈が第一の入口

オンボーディングはユーザーが最初に見る画面であり、「このアプリは誰のためのものか」を決定する。ここで「亡くなったペットのためのアプリ」と感じさせると日常ユーザーは即離脱する。

## 期待する動作

**3スライド構成（スキップ可）:**

- スライド1: 「ペットとの毎日を、ずっと残せる場所。」  
  → 写真を撮る・記録するイメージ（生きているペットの自然な日常写真）

- スライド2: 「続けるほど、大切な物語になる。」  
  → タイムラインに積み重なった記録のイメージ

- スライド3: 「いつかのあなたを、今日の記録が支える。」  
  → ロスケアを示唆するが、前面に出さない静かな表現

**禁止表現:**
- 「亡くなった」「別れ」「グリーフ」「ペットロス」「虹の橋」
- 「ゆっくりで大丈夫です」（ロス後向けに聞こえる）

**CTA:** 「はじめる」（大きく）/ 「スキップ」（小さいテキストリンク）

## 実装方針

`onboarding.tsx` の文言・スライド構成を全面書き直し。  
ロジックの変更なし。コピー・ビジュアルのみ変更。
