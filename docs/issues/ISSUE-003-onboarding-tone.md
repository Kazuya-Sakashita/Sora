# ISSUE-003: オンボーディングがペットロス専用アプリに見える可能性

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | open |
| 関連画面 | オンボーディング画面 |
| 関連ファイル | `components/screens/onboarding.tsx` |

## 問題の概要

現在のオンボーディングの文言・雰囲気が、生前から使うアプリとして自然に入れるかを確認・修正する必要がある。

## 詳細

仕様書の最重要方針の一つが「生前から自然に使える・ペットロス専用に見せない」こと。オンボーディングはユーザーが最初に見る画面であり、ここで「亡くなったペットのためのアプリ」と感じさせると離脱する。現在の実装を確認し、必要なら修正する。

## 期待する動作

- 「ペットとの毎日を残す場所」として伝わる
- 「亡くなった後のために」という雰囲気が出ていない
- ポジティブで温かい第一印象
- 「はじめる」CTAへの心理的ハードルが低い

## 確認・修正ポイント

1. タイトル・サブコピーの文言
2. 背景・カラーの雰囲気（暗すぎないか）
3. CTAボタンのラベル
4. スライド枚数と各スライドのメッセージ

- [ ] 既存機能に影響なし確認済み
