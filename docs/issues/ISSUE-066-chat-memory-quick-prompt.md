# ISSUE-066: 直近記録をチャットのクイックプロンプトに反映する

name: ISSUE-066
type: improvement
priority: High
status: done

---

## 背景

Soraのコアバリューは「記録を通して、もう一度つながれる体験」。しかし現在のチャット画面のクイックプロンプトは「会いたいな」「今日は少しつらい」などの固定文言で、ユーザーの実際の記録と接続されていない。記録とチャットが独立した体験になっており、「この記録をAIに話したい」という自然な流れが生まれていない。

## 問題

`quickPrompts` は固定配列で、ユーザーの記録データを参照していない。記録を重ねているユーザーも、重ねていないユーザーも同じプロンプトが表示される。「自分のペットの話」を始めるための入口がない。

## 目的

直近の記録タイトルを使ったクイックプロンプトを1つ動的に追加し、記録→会話の自然な流れを作る。記録がない場合は従来の固定プロンプトのみを表示する。

## スコープ

- `useApp()` から `memories` を取得
- 直近1件の記録がある場合: `「${memories[0].title}」のこと、話してもいい？` をクイックプロンプトの先頭に追加
- 記録がない場合: 従来の固定プロンプトのみ
- プロンプトが長い場合は先頭20文字で切り詰め + `…`
- 最大5プロンプト表示（動的1 + 固定4 = 5、または固定4のみ）

## やらないこと

- APIを使った動的プロンプト生成
- 記録内容（本文）のクイックプロンプトへの反映
- alive期の動的プロンプト（RAINBOW_BRIDGE のみ有効）

## 受け入れ条件

- [ ] 記録がある場合、直近記録タイトルを使ったクイックプロンプトが先頭に表示される
- [ ] 記録がない場合、従来の固定プロンプトのみ表示される
- [ ] クイックプロンプトを押すとその文言でメッセージが送信される
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過
- [ ] gstackで動作確認済み
