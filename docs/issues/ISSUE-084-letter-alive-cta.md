# ISSUE-084: ことば画面のalive期プレースホルダーを記録継続動機に接続する

name: ISSUE-084
type: improvement
priority: High
status: open

---

## 背景

ISSUE-071でことば画面のalive期ガードを実装した。現在のプレースホルダー:

```tsx
// letter.tsx (alive期)
<h2>まだこのことばはないかもしれない</h2>
<p>ことばは、記録を積み重ねた先に生まれます。今は、毎日のできごとを残していく時間です。</p>
<button>記録を見る</button>  // → timeline へ
```

倫理的に正しい表示だが、ユーザーにとって「拒否されて終わり」の体験になっている。「ことば」タブを開いた alive 期ユーザーは、「記録を続けることでいつかここに手紙が届く」という forward-looking な期待値を持っていない。

また「記録を見る」ボタンは timeline（過去の記録一覧）に遷移するが、タブを訪問したユーザーのニーズは「今から記録を増やしたい」ではなく「この機能に興味を持った」だ。

## 問題

```tsx
// 現在: 拒否 → timeline への誘導（ニーズとミスマッチ）
// 改善後: 説明 → 記録追加への誘導（ニーズに合致）
```

## 目的

alive期ユーザーが「ことば」タブを訪問した瞬間を、「記録を続けることの意義」を再確認する機会に変える。

## スコープ

- `letter.tsx` のalive期プレースホルダーUIを更新:
  - タイトル: 「まだこのことばはないかもしれない」→ 変更なし（正確な表現として維持）
  - 説明文: 現状の文章の後に1文追加:
    - 現在: 「ことばは、記録を積み重ねた先に生まれます。今は、毎日のできごとを残していく時間です。」
    - 追加: 「{pet.name}との記録が積み重なったとき、Soraがあなたへの言葉を作ります。」
  - CTAボタン:
    - 現在: 「記録を見る」→ timeline
    - 変更後: 「今日の記録を残す」→ timeline に遷移（setPendingMemoryTitle は不要、timelineに遷移するだけ）
  - ボタンテキストの変更でユーザーのアクション意図を「見る」→「残す」に揃える

## やらないこと

- ガード条件の変更（rainbow_bridge 以外は引き続きブロック）
- プレビュー手紙の生成・表示
- alive期ユーザーへの部分的な機能開放

## 受け入れ条件

- [ ] alive期プレースホルダーに「{pet.name}との記録が積み重なったとき、Soraがあなたへの言葉を作ります。」が表示される
- [ ] CTAボタンが「今日の記録を残す」に変更されている
- [ ] ボタンタップでtimeline画面に遷移する
- [ ] rainbow_bridge期には通常のことば生成UIが表示される
- [ ] `pnpm tsc --noEmit` 通過
- [ ] `pnpm test` 全通過

## 確認観点

- ユーザーの心を傷つけない
- Soraらしさを壊していない（過度な期待値を作らない）
- AI依存を強めすぎていない
- 既存機能を壊していない
