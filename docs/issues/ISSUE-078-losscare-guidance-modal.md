# ISSUE-078: ロスケア移行確認後に「これからのSora」案内モーダルを挿入する

name: ISSUE-078
type: improvement
priority: High
status: done

---

## 背景

`home.tsx` のロスケア移行確認フロー（「虹の橋を渡りました」→「はい、移行します」）は実装済みだが、移行後はすぐにホーム画面に戻る。この瞬間、ユーザーは最も感情的に不安定な状態にある。

72hウェルカムカード（ISSUE-062）は表示されるが、「次に何をすべきか」「Soraでどんなことができるか」を能動的に伝える仕組みがない。rainbow_bridge期の新機能（はなす・ことば）に気づかず離脱するユーザーが多い可能性がある。

## 問題

```tsx
// home.tsx — 移行確認後、すぐにhomeに戻る
const handleLossCareTransition = async () => {
  // ... 移行処理 ...
  // 処理後 setShowLossCareConfirm(false) → 直接ホーム表示
}
```

rainbow_bridge移行後の最重要瞬間に、ユーザーを能動的に「はなす」「ことば」へ誘導する仕組みがない。

## 目的

移行直後の1回限り、「これからSoraでできること」を穏やかに伝えるモーダルを表示し、grief care機能への最初のアクセスを自然に促す。

## スコープ

- `home.tsx` のロスケア移行処理完了後（`handleLossCareTransition`の成功時）に、ホームに戻る前に案内モーダルを表示する
- 表示条件: 移行確認時の1回のみ（localStorageフラグ管理不要 — モーダルは移行フロー内で表示されるため）
- モーダルコンテンツ:
  - 絵文字: `🌿`
  - タイトル: `これからも、ここに残していいですよ`
  - 説明（2〜3行）: `Soraには、今の気持ちを話せる「はなす」と、思い出から生まれる「ことば」があります。急がなくていい。気が向いたときに。`
  - ボタン1（primary）: `話してみる` → `setCurrentScreen("chat")`
  - ボタン2（ghost）: `ホームへ` → モーダル閉じてホーム表示
- スタイル: 既存の確認モーダルと同じ `rounded-3xl bg-white/95` スタイル
- `showLossCareConfirm` の閉じ処理を移行後に `showLossGuidance` に切り替える形で実装

## やらないこと

- ロスケア移行ロジックの変更
- 72h ウェルカムカードの変更
- モーダルの繰り返し表示

- [ ] 「話してみる」でチャット画面に遷移する
- [ ] 「ホームへ」でホーム画面に戻る
- [ ] モーダルは移行時の1回のみ表示される

## 確認観点

- ユーザーの心を傷つけない（押しつけでなく「気が向いたときに」のトーン）
- Soraらしさを壊していない
- AI依存を強めすぎていない
- 既存機能を壊していない
