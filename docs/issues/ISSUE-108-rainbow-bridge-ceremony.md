# 概要

ペットステータスをALIVE → RAINBOW_BRIDGEに変更する際の体験を、「ボタンを押す」から「大切な一線を越える儀式」として設計し直す。変更後72時間の体験も、よりパーソナルで受容的にする。

# 背景

テストユーザーから「Rainbow Bridge移行後にどうしていいかわからなかった」という声があった。現在のモーダルは「本当によいですか？」という確認と「変更する」ボタンで終わる。これは機能的には正しいが、感情的には「急に変わった」という感覚を生む。

また、移行後のLoss Welcome Cardはあるものの、ペット名が含まれておらず、何件記録したかへの言及もなく、「一般的な慰め」に見える。

# 解決したい課題

- 移行が「設定変更」として処理されてしまう
- 移行後の「あなたの記録はここにある」という安心感が届いていない
- 72時間以内の体験が汎用的（ペット名・記録数への言及なし）
- 移行後すぐに「機能が変わった」ことへの戸惑い

# ユーザー価値

「ここに来ていいんだ」「急がなくていい」という許可を、アプリ全体が与え続けてくれる安心感。最初の数日が、長期継続の基盤になる。

# 実装方針

**移行モーダルのリデザイン（settings.tsx）:**

現在:
「虹の橋へ旅立ったことを記録しますか？」→「記録する」ボタン

改善後:
1. 確認画面：「{name}とのお別れを、ここに残しておいてもいいですか」
   - サブコピー：「{name}との{n}日間の記録が、ここにあります」
   - ボタン：「残しておく」「まだいい」
2. 変更完了画面（新規）：
   - 「{name}との{n}日間が、ここに残りました」
   - 「いつでも来てください。急がなくていいですよ」
   - 背景は静かなグラデーション（空・夕暮れ系）
   - 「ホームへ」ボタン（3秒後に自動推移でもよい）

**移行後72h以内のLoss Welcome Card改善（home.tsx）:**

現在：
「急がなくていい。泣いていい。ここにいていい。」（汎用）

改善後：
「{name}のことを、{n}件記録してきましたね。その全部が、今もここにあります。急がなくていいですよ。」

**移行後7日・30日のカード（home.tsx）:**

現在：7/14/30日後のTimelineCareCardがある

改善後：
- 7日：「1週間経ちましたね。今日の気持ちは、どうですか」
- 30日：「1ヶ月経ちました。{name}の写真を、ゆっくり見返してもいい日かもしれません」
- 3ヶ月：「{n}日が経ちました。{name}の記録が、ずっとここにいます」

**「今日はここにいていい」の繰り返し:**
- ホームを開くたびに1回だけ、ランダムで「{name}のことを思い出す日があってもいい」系のやさしい一言を表示（Rainbow Bridge期限定、3ヶ月間）

# 受け入れ条件

- [ ] 移行確認モーダルに記録件数・日数が表示される
- [ ] 移行完了画面が新設され、「残りました」という安心感がある
- [ ] Loss Welcome Cardがペット名と記録件数を含む文章になっている
- [ ] 7日・30日・3ヶ月のカードが時間経過と記録に言及する内容になっている
- [ ] 移行後90日間、ホームに「一緒にいていい」を感じさせる言葉が小さく添えられる

# やらないこと

- 「立ち直れましたか？」「前を向けてきましたか？」という問いかけ
- 「専門家に相談を」という強い誘導（さりげなくリンクがある程度）
- 泣き顔・暗いイメージの絵文字（静けさを意識する）
- 移行を「後悔」させる演出（「本当に？」の強調）

# 検証方法

- 移行後7日間のアプリ起動率
- 移行後30日間のRetention率
- Loss Welcome Card の「閉じずに読んだ」率（滞在時間計測）
