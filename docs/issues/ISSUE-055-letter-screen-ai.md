# ISSUE-055: 「ことば」画面をAIレターに置き換える

name: ISSUE-055
type: feature
priority: High
status: done

---

## 概要

ロスケアモード専用の「ことば」画面（`letter.tsx`）が5パターンのhardcodedランダム文を返すだけになっている。AIチャット（ISSUE-054）と同じく Claude API による文脈を踏まえた返答に置き換える。

## 背景

`letter.tsx` は ISSUE-009 で「Phase 3まで非表示」とされていたが、ISSUE-033 でロスケアモード（rainbow_bridge）時に表示を解放した。しかし中身は実装されないまま放置されており、5パターンの汎用フレーズをランダム返却するだけ。AIチャット（ISSUE-054）と並ぶロスケアの主要機能なのに品質差が大きく、離脱の原因になっている。

また Settings の月次レター（ISSUE-051）と「ことば」の役割が未整理で、ユーザーが混乱する。

## 目的

- ロスケアユーザーに「自分とこの子のことを理解してくれている」と感じさせる手紙を届ける
- チャット（対話）とレター（手紙）を役割で使い分けられるようにする
  - チャット: 今の気持ちをその場で吐き出す
  - ことば: 読んで心に残す、保存する

## スコープ

- `POST /api/pets/[petId]/letter/generate` を新規追加
  - リクエスト: なし（パスパラメータのみ）
  - レスポンス: `{ content: string }`
  - ペット名・直近の記録5件・ペットのステータスをシステムプロンプトに含める
  - 150〜200字・手紙形式・汎用文禁止・絵文字禁止
- 「もう一度読む」ボタンで再生成（生成中はローディング表示）
- 生成した手紙をローカルに最大5件保存（`savedLetters` stateで管理）
- 月次レター（Settings）への導線テキストを画面下部に追加：「毎月の記録から作る手紙は → 設定から見る」
- レート制限: 1ユーザー・1分間に5回まで（再生成の連打を防ぐ）
- エラー時はインラインメッセージ

## 非スコープ

- 手紙のDB永続化（月次レターがその役割を担う）
- 手紙のシェア機能

## API変更

- あり
- `POST /api/pets/[petId]/letter/generate` を openapi.yaml に追加
