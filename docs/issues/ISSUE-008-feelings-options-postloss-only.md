# ISSUE-008: 気持ち記録の選択肢がロス後ユーザー専用でしか使えない

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | open |
| 関連画面 | 気持ち記録画面 |
| 関連ファイル | `components/screens/feelings.tsx` |

## 問題の概要

現在の気持ち選択肢4つがすべてロス後の感情で、生きているペットと暮らすユーザーが選べるものがない。

## 詳細

```tsx
// 現状の feelingOptions（問題）
{ emoji: "😢", label: "かなしい" }       // ロス後
{ emoji: "💭", label: "会いたい" }       // ロス後（亡くなった子に会いたい）
{ emoji: "🌿", label: "少し落ち着いている" } // ロス後
{ emoji: "💝", label: "ありがとうと思えた" } // ロス後
```

ペットが元気で生きているユーザーがこの画面を開いたとき、選べる選択肢が1つもない。「かなしい」「会いたい」は日常の感情としては使えず、アプリの世界観が「ロス後専用」と感じられる。

## 期待する動作

生前・ロス後どちらのユーザーにも使える感情タグにする。

```tsx
// 修正案（生前 + ロス後どちらにも対応）
{ emoji: "🥰", label: "うれしい" }
{ emoji: "😌", label: "おだやか" }
{ emoji: "💕", label: "愛おしい" }
{ emoji: "😅", label: "笑えた" }
{ emoji: "😢", label: "さみしい" }
{ emoji: "😟", label: "心配" }
```

openapi.yamlのFeelingTagとも整合させること。

## 実装方針

`feelings.tsx` の `feelingOptions` 配列を差し替える。ラベルの変更なのでデータ型の変更は不要。
