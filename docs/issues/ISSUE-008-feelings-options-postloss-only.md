# ISSUE-008: 気持ち記録の選択肢がロス後ユーザー専用でしか使えない

## 基本情報

| 項目 | 内容 |
|------|------|
| 優先度 | high |
| ステータス | done |
| 関連画面 | 気持ち記録画面 |
| 関連ファイル | `components/screens/feelings.tsx` |

## 問題の概要

現在の気持ち選択肢4つがすべてロス後の感情で、生きているペットと暮らすユーザーが選べるものがない。

## 詳細

```tsx
// 現状の feelingOptions（問題）
{ emoji: "😢", label: "かなしい" }          // ロス後
{ emoji: "💭", label: "会いたい" }          // ロス後（亡くなった子に会いたい）
{ emoji: "🌿", label: "少し落ち着いている" } // ロス後
{ emoji: "💝", label: "ありがとうと思えた" } // ロス後
```

ペットが元気で生きているユーザーがこの画面を開いたとき、選べる選択肢が1つもない。

## 仕様変更による修正内容の更新（2026-04-28）

MVP再定義（`docs/specs/sora-product-rethinking.md`）で感情タグを **5種類** に定義した。

```tsx
// 修正後（生前・ロス後どちらにも対応・5種類）
{ emoji: "🥰", label: "うれしい" }
{ emoji: "😌", label: "おだやか" }
{ emoji: "😄", label: "笑った" }
{ emoji: "😟", label: "心配" }
{ emoji: "💕", label: "愛おしい" }
```

旧案（6種類）から「さみしい」を削除し、「笑えた」→「笑った」に統一した。  
openapi.yaml の FeelingTag とも整合させること。

## 実装方針

`feelings.tsx` の `feelingOptions` 配列を5種類に差し替える。ラベルの変更のみでデータ型の変更は不要。
