# ISSUE-041: Sora+ 機能プレビュー（価値の先見せ）

name: ISSUE-041
type: feature
priority: High
status: open

---

## 概要

Free ユーザーがフォトブック・年次レポートボタンをタップした際、即 402 ではなくサンプル出力のプレビュー画像を見せてから UpgradeModal を表示する。

## 背景

現状は「タップ → 402 → UpgradeModal」の順で、ユーザーは何が得られるか知らずに課金を求められる。「払う前に価値が分からないものには払わない」のが標準行動。プレビューがあれば転換率が上がる。

## 目的

- Plus 機能の価値を視覚で伝え、Free→Plus 転換率を向上させる

## スコープ

- `public/samples/` に静的サンプル画像を配置
  - `photobook-sample.png`：フォトブックPDFの1ページ目スクリーンショット（ダミーデータ）
  - `report-sample.png`：年次レポートの表紙スクリーンショット（ダミーデータ）
- `timeline.tsx` のフォトブックボタン・レポートボタン：402 時に UpgradeModal の前にプレビューモーダルを1ステップ挟む
  - プレビューモーダル構成：サンプル画像 + 「Sora+ で作成できます」コピー + 「アップグレードする」ボタン
- UpgradeModal 自体は変更しない

## 非スコープ

- 動的サンプル生成（ユーザーの実データを使ったプレビュー）
- フォトブック・レポート以外の機能へのプレビュー追加

## API変更

- なし（静的ファイルのみ）

## 受け入れ条件

- [ ] Free ユーザーがフォトブックボタンをタップするとサンプル画像が表示される
- [ ] 「アップグレードする」でUpgradeModalに遷移する
- [ ] Plus ユーザーはプレビューをスキップして直接ダウンロードが始まる
- [ ] `pnpm tsc --noEmit` 通過

## テスト観点

- 正常系: Freeユーザーにプレビューモーダルが表示される
- 正常系: Plusユーザーにはプレビューが表示されない
- UI動作: プレビュー → アップグレード → モーダルの遷移
