# 概要

現在の気持ちタグ（5個の選択式）に加え、「今日の気持ちをもう少し書く」という任意のフリーテキスト入力を追加する。タグでは収まらない感情を、Soraに届けられるようにする。

# 背景

ペットロスの感情は「悲しい」「つらい」「よくわからない」というタグでは表現しきれない。「写真を見ていたら急に泣けてきた」「今日は不思議と平気だった、なぜか罪悪感がある」「怒ってしまった、なぜ怒ってしまったのか」という複雑さがある。

タグによる感情分類は「整理」させてしまい、感情と「一緒にいる」体験にならない。感情に名前をつけることを急かすのではなく、書けたら書ける余白が必要。

# 解決したい課題

- 5個のタグに収まらない感情が「表現できない」ままになる
- タグを選ぶこと自体がハードルになる日がある
- Rainbow Bridge期に「気持ちを誰かに届けたい」のに届け先がない

# ユーザー価値

「悲しい」より詳しく書けた日、その言葉をAIが受け取ってくれた体験。「言語化できた」という小さな安心感。

# 実装方針

**UI変更（feelings.tsx / quick-record-sheet.tsx）:**
- タグ選択の下に「もう少し書く（任意・50文字）」テキストエリア
- placeholder: 「今日の気持ち、少し書いておきますか」
- 入力なしでも保存可能（完全任意）

**API変更:**
- `POST /api/pets/{petId}/feelings` に `memo?: string` フィールドを追加（既にスキーマ上はある可能性）
- AI反応プロンプトにこのメモを含める

**Rainbow Bridge期の専用エントリー:**
- ホーム画面に「今日の気持ちを残す」ボタン（小さく、控えめに）
- タップ → タグ選択 + フリーテキスト → 保存 → AI受け取り反応
- 記録（写真・タイトル）とは別の「感情の記録」として位置づける

# 受け入れ条件

- [ ] タグ選択画面にフリーテキスト入力欄が追加されている
- [ ] 入力は任意で、書かなくても保存できる
- [ ] 入力したテキストがAI反応プロンプトに含まれる
- [ ] Rainbow Bridge期のホームに「今日の気持ちを残す」エントリーがある
- [ ] 入力内容がタイムラインの気持ちタグ表示に付随して見返せる

# やらないこと

- 感情分析・スコアリングの表示（「あなたは悲しみ段階の第2フェーズです」などは禁止）
- 入力内容の統計化・可視化（プライベートなメモとして扱う）
- 入力を強制するUI（「今日の気持ちを入力してください」という催促）

# 検証方法

- フリーテキスト入力率（気持ちタグ使用者の中で入力した割合）
- フリーテキストを入力した日の翌日Retention率 vs 未入力の日
- 定性：入力内容の長さ分布（書けるとどれくらい書くか）
