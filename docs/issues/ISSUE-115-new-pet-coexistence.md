title: 新しい子と先代の共存設計
type: ux
priority: medium
status: 未対応
---

# 概要

新しいペットを迎えた後も、先代ペットの記録が自然に共存できるよう、マルチペット体験を設計し直す。先代の記録が「過去のもの」として押し込まれず、今も大切にできる場所であり続ける。

# 背景

虹の橋を渡ったペットの後に、新しいペットを迎えるユーザーがいる。現在のUIは「アクティブなペット」中心設計のため、Rainbow Bridge期のペットが「過去の存在」として扱われてしまう。「新しい子を迎えても、ポチのことを忘れたくない」というユーザーの気持ちに応える設計が必要。

# 解決したい課題

- 新しいペットを追加すると、先代のペットが埋もれる
- Rainbow Bridge期のペットと現役ペットを同等に扱う選択肢がない
- 「先代の記録を見返す」動線が新しいペット登録後に弱くなる

# ユーザー価値

「新しい子を迎えても、先代のことをちゃんと大切にできている」という安心感。先代と新しい子が同じ「家族」として共存できる体験。

# 実装方針

**ペット切り替えUI（ホーム・ナビ）:**
- 現在：1ペット中心の単一ビュー
- 変更：ペットアバター列（最大5頭）でタブ切り替え
- Rainbow Bridge期ペットは薄いボーダー or 小さいリボンで区別（過剰演出しない）

**「先代の子」セクション（ホーム）:**
- Alive期のペットが存在する場合、Rainbow Bridge期ペットを「大切な記憶」セクションとして下部に表示
- 「{name}の記録を見る」リンクで先代のタイムラインへ

**ペット登録フロー変更:**
- 新しいペットを追加する際、既存Rainbow Bridge期ペットの扱いを確認
- 「ポチのことはこれからも大切に記録します」という確認文（強制ではなく自然な確認）

**データ変更:**
- Pet.role の `rainbow_bridge` と `active` の混在を正しくサポート
- API: `GET /api/pets` で全ペット返却（RB期も含む）

# 受け入れ条件

- [ ] 複数ペット登録時にペット切り替えタブが表示される
- [ ] Rainbow Bridge期ペットがホームから1タップでアクセスできる
- [ ] 新しいペット追加後も先代の記録が同じアクセシビリティで閲覧できる
- [ ] ペットアバター列でRainbow Bridge期ペットが視覚的に区別されている

# やらないこと

- Rainbow Bridge期ペットを別アプリ・別アカウントに分離する
- 先代ペットの記録を「アーカイブ」として隠す
- 新しいペット追加を制限する

# 検証方法

- マルチペットユーザーの先代ペットタイムライン閲覧率
- 新しいペット追加後のRainbow Bridge期ペット記録投稿率
- マルチペットユーザーの月次Retention
